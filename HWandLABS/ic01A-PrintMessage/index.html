Escape Sequences

Escape sequences are a way to represent "difficult" characters inside your source code. For instance, if you want to create a String that contains a newline character, you can't just press the Enter key when typing in your code. The Enter key is a control character, and, instead of inserting a newline into your document, it issues the "newline command" to your editor.

The same kind of problem occurs when you want to display a double quote or a backslash inside a String. You can find out how to write escape sequences your online class notes. You can go back to that section if you need more help.

Here are some things you should notice:

  • The first sentence of the output is surrounded with double quotes. You'll need to use the escape sequence to embed a quote for that to work.
  • There are also two embedded quotes in the second sentence.
  • The third line is blank. You can use System.out.println();, with no arguments, to display a blank line.
  • The last line has several backslashes. However, since the backslash is the escape character in Java, you need to "escape the escape character" to produce a literal backslash. All that really means is that you need two backslashes for every one that appears in output.
