Swap Ends


Array parameters are different than primitive parameters, because array parameters are references. Even though the parameter is passed "by value", the value (or array reference), still refers to the original array that was created outside of the method.

That means that array parameters can be used as function outputs. By using output parameters, even a void procedure can return information to its caller.

Swap Ends

Write the procedure named swapEnds(). You'll find the file (SwapEnds.java) in this week's code folder.) Here are the specifications:

The swapEnds() procedure is passed an array of ints named nums. You should swap the first and last elements in the array. Since this is a procedure, you won't have any return value. (The array length will be at least 1.)

Here are the Javadoc tags:

  • @param nums an array of int to be modified.
  • No @return because this is a procedure.

Here are some examples:

  • swapEnds({1, 2, 3, 4}) changes nums to {4, 2, 3, 1}
  • swapEnds({1, 2, 3}) changes nums to {3, 2, 1}
  • swapEnds({8, 6, 7, 9, 5}) changes nums to {5, 6, 7, 9, 8}
