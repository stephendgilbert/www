Lab 18

Output Formatting


Your third programming exam involves parsing and formatting, converting between Strings and different kinds of numbers. The Lab problems today looks at using String.format() to format output for printing. (String.format() produces a String which can then be printed or used in a GUI program; System.out.printf() only prints output on the console.

L18: Produce Bill

Create a new standard console application named ProduceBill. Place it in the Chapter04 folder, since that's where the test programs are located.

  1. Prompt the user to enter weight and price (in that order). Use a Scanner to initialize the variables.
  2. Calculate the totalPrice using weight and price as shown below.
  3. Use output formatting to produce results like this: 
        Unit Price:     $4.25 per pound
    Weight:          2.56 pounds
    TOTAL:         $10.89

Here are the specific things to watch out for:

  • There are 4 spaces at the beginning of each line of output. These are not TAB characters.
  • The labels are left-aligned in a field 11 wide.
  • The values are right-aligned in a field 10 wide.

The really tricky part here, though is the floating dollar sign. To get the floating dollar sign, you have to first format the amounts without the width specifier, and then format the resulting Strings with a width specifier.

Here, for instance, is what the output should look like with larger numbers: 
    Unit Price:    $27.95 per pound
    Weight:        123.00 pounds
    TOTAL:      $3,437.85
