Lab 33, 34 & 35

Multi-way Branches


You can combine decisions by using the logical operators, as you saw in the last set of lab exercises. You can do something similar by implementing multi-way branches, using sequential if statements, or by nesting. To get started, make sure you have read Section 6.5—Multiple Choice, and completed both H18—Date Fashion and H19—Parrot Trouble. That will prepare you to work on these problems.

Complete these problems using sequential or nested if statements. Do not use the logical operators, like && or ||.

L33—Evenly Spaced

Write the function named evenlySpaced(). Here are the specifications:

Given three ints, a b c, one of them is small, one is medium and one is large. Return true if the three values are evenly spaced, so the difference between small and medium is the same as the difference between medium and large.

Here are the Javadoc tags:

  • @param a the first number.
  • @param b the second number.
  • @param c the third number.
  • @return true if the numbers meet the conditions described.

Here are some examples:

  • evenlySpaced(2, 4, 6) returns true
  • evenlySpaced(4, 6, 2) returns true
  • evenlySpaced(4, 6, 3) returns false

L34—Lucky Sum

Write the function named luckySum(). Here are the specifications:

Given 3 int values, a b c, return their sum. However, if one of the values is 13 then it does not count towards the sum and values to its right do not count. So for example, if b is 13, then both b and c do not count.

Here are the Javadoc tags:

  • @param a the first number.
  • @param b the second number.
  • @param c the third number.
  • @return the lucky sum as calculated here.

Here are some examples:

  • luckySum(1, 2, 3) returns 6
  • luckySum(1, 2, 13) returns 3
  • luckySum(1, 13, 3) returns 1

L35—Making Bricks

Write the function named makeBricks(). Here are the specifications:

We want to make a row of bricks that is goal inches long. We have a number of small bricks (1 inch each) and big bricks (5 inches each). Return true if it is possible to make the goal by choosing from the given bricks. This is a little harder than it looks and can be done without any loops.

Here are the Javadoc tags:

  • @param small number of small bricks.
  • @param big number of big bricks.
  • @param goal number of bricks in desired row.
  • @return true if possible with available bricks.

Here are some examples:

  • makeBricks(3, 1, 8) returns true
  • makeBricks(3, 1, 9) returns false
  • makeBricks(3, 2, 10) returns true
