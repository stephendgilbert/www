Lab 46, 47 & 48

More Complex Loops


Before the midterm, you had several lab assignments using the for and while loops to repeat a set of actions. Now, you've added sentinel, flag-controlled and different jump statements to the mix.

For this lab, we'll solve three more complex loop problems. You can complete these problems using any loops you like. You may need multiple loops. You may also make use of the the different jump statements, ike break and continue.

Solve these programs with loops, not with regular expressions and the replaceAll() method.

L46—Without String

Write the function named withoutString(). Here are the specifications:

Given two strings, base and remove, return a version of the base string where all instances of the remove string have been removed (not case sensitive). You may assume that the remove string is length 1 or more. Remove only non-overlapping instances, so with "xxx" removing "xx" leaves "x".

Here are the Javadoc tags:

  • @param base the base String to modify.
  • @param remove the substring to remove.
  • @return the base String with all copies of remove removed.

Here are some examples:

  • withoutString("Hello there", "llo") returns "He there"
  • withoutString("Hello there", "e") returns "Hllo thr"
  • withoutString("Hello there", "x") returns "Hello there"

L47—Mirror Ends

Write the function named mirrorEnds(). Here are the specifications:

Given a String str, look for a mirror image (backwards) string at both the beginning and end of the given string. In other words, zero or more characters at the very beginning of the given string, and at the very end of the string in reverse order (possibly overlapping). For example, the string "abXYZba" has the mirror end "ab".

Here are the Javadoc tags:

  • @param str the String to search.
  • @return the string that is mirrored at the end.

Here are some examples:

  • mirrorEnds("abXYZba") returns "ab"
  • mirrorEnds("abca") returns "a"
  • mirrorEnds("aba") returns "aba"

L48—Don't Worry; G Happy!

Write the function named gHappy(). Here are the specifications:

We'll say that a lowercase 'g' in a string is "happy" if there is another 'g' immediately to its left or right. Return true if all the g's in the given string are happy.

Here are the Javadoc tags:

  • @param str the String to search.
  • @return true if all lowercase 'g's are happy.

Here are some examples:

  • gHappy("xxggxx") returns true
  • gHappy("xxgxx") returns false
  • gHappy("xxggyygxx") returns false
