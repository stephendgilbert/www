Section 11.5

Array Review Sheet


Now that we've covered most of the basic array topics, let's review what you've learned.

Array Basics

An array is a collection of variables, all of which have the same type (int, double, String and so forth). The individual variables in the collection are accessed by using an array variable along with a subscript or index number, indicating the variable's position inside the collection.

Here is the basic syntax you should know:

  • Create an array variable: double[] a;
  • Instantiate array elements: a = new double[5];
  • Define and instantiate: int[] b = new int[7];
  • Instantiate and initialize an array: String[] words = {"one", "two", "three"};
  • Access first element of an array: String first = words[0];
  • Access last element of an array: String last = words[words.length - 1];
  • See if array has any elements: if (b.length == 0) ... // empty array

Arrays and Methods

You cannot pass an array to a method or return an array from a method. You can, however, pass and return array variables. When you pass an array variable to a method, the method can change the elements in the original array that the variable refers to. To avoid this, you can copy the elements in the array.

Here is the basic syntax you should know:

  • Define an array formal parameter: void printArray(int[] a) ...
  • Define an array return type: int[] makeArray(int n) ...
  • Pass an array to a method: int[] a = {1, 2, 3}; printArray(a);

Arrays and Loops

Normally you will use the for loop, (or, the enhanced, for-each version of the for loop), to process the elements in an array. Because arrays are fixed-length, you can use the length field (not the length() method, as with Strings) to act as the loop bounds.

Here are the common modifying loops you should know how to write:

  • Initialize the elements in an array. (Assume a is an array and that value has been defined.):
    for (int i = 0; i < a.length; i++)
        a[i] = value;
  • Modify the elements in an array. (Assumes b is an array of double and that percent is the percentage to change each element by.):
    for (int i = 0; i < b.length; i++)
        b[i] *= (1 + percent); // or: b[i] = b[i] * (1 + percent)
  • Move or shift the elements in an array. (Assumes words is an array of String. Shifts each string to the left. Notice loop goes to words.length - 1, not words.length):
    String temp = words[0]; // save first word
    for (int i = 0; i < words.length - 1; i++)
        words[i] = words[i + 1]; // copy next word into current
    words[words.length - 1] = temp; // copy first word to end

Non-Modifying Loop Algorithms

Some algorithms don't need to change the elements in the array. For these algorithms, you can use either the for loop, or the enhanced version of the for loop. I'll show you both.

There are the three things you should know how to do: count, sum (or average), and find the extreme values in the array. Let's look at those.

Counting Elements

For counting elements, you'll normally need to count based on some condition. For instance, you'll count all of the even numbers, or all of the odd numbers, or all of the 2s that are not followed by a 3.

Here are two examples of the basic skeleton for this kind of loop. This example uses an int array named a and it counts the odd numbers.

  • Using a regular for loop, the code looks like this:
    int counter = 0; // always an int, always set to 0
    for (int i = 0; i < a.length; i++)
        if (a[i] % 2 == 1) // whatever condition you want to count
            counter++; // add 1 to the counter
    Using the enhanced for loop, the same problem would look like this:
    int counter = 0; // always an int, always set to 0
    for (int num: a) // no index. num is current element
        if (num % 2 == 1) // whatever condition you want to count
            counter++; // add 1 to the counter

Summing or Averaging Elements

The code for summing elements is almost the same as counting them. When you are summing, you'll use an accumulator instead of a counter, and inside your loop, you'll add the value of the element to the accumulator, instead of adding one to the counter. As with counting, problems will normally ask you to sum values based on some condition. An example would be summing all purchases less than ten dollars.

Let's try that. Here are two examples of the basic skeleton for this kind of loop. This example uses an double array named purchase and it sums the purchases less than ten dollars.

  • Using a regular for loop, the code looks like this:
    double amount = 0.0; // type is same as element type
    for (int i = 0; i < purchases.length; i++)
        if (purchases[i] < 10.0) // whatever condition you want to count
            amount += purchases[i]; // or amount = amount + purchases[i]
    Using the enhanced for loop, the same problem would look like this:
    double amount = 0.0; // type is same as element type
    for (double num: purchases) // no index. num is current element
        if (num < 10.0) // whatever condition you want to count
            amount += num; // or amount = amount + num

Extreme Values

Extreme values just means finding the largest or smallest element. The algorithm for this is fairly straightforward, and you should memorize it:

  1. Store the first element in the array in a variable (largest or smallest or both).
  2. Loop through the remaining elements in the array. (You can actually visit the first element once again, if you want to use the enhanced for loop.) At each position in the loop:
    1. Compare the current array element to your saved value.
    2. If the current element is smaller than your (smallest) variable, save the current element in smallest.
    3. If the current element is larger than your (largest) variable, save the current element in largest.

At the end of your loop, the largest or smallest variable will contain the appropriate value.

Here's an example that finds the smallest word in an array of String named words:

  • Using a regular for loop, the code looks like this:
    String smallest = words[0]; // smallest so far
    for (int i = 1; i < words.length; i++)
        if (words[i].length() < smallest.length())
            smallest = words[i]; // new smallest word
    Using the enhanced for loop, the same problem would look like this. (Actually looks at words[0] twice, so is not quite as efficient.)
    String smallest = words[0]; // smallest so far
    for (word: words)
        if (word.length() < smallest.length())
            smallest = word; // new smallest word

Object and 2D Arrays

The only difference between primitive (int, double, etc.) arrays and object arrays, is that you need to remember to separately instantiate the array elements; the elements are not automatically instantiated like the primitive array values are.

Here are some examples using Strings and GImage objects:

  • Create a String array: String[] words = new String[10];
    Now elements contain null, not String objects.
  • Instantiate array elements using a Scanner:
    for (int i = 0; i < words.length; i++) words[i] = cin.nextLine();
  • Instantiate array elements using a literal (empty String):
    for (int i = 0; i < words.length; i++) words[i] = "";
  • Create an array of GImage references, and then initialize using a loop and the same file for each image:
    GImage[] images = new GImage[5];
    for (int i = 0; i < images.length; i++)
        images[i] = new GImage("clown.jpg");

2D Arrays

2D arrays represent a grid or matrix, like a spreadsheet. With a 2D array, you have a separate bracket for the row, followed by a bracket for the column. As with 1D arrays, both row and column start with 0.

With a 2D array variable, the length field represents the number of rows in the array. To get the number of columns in any individual row, you need to use the length field for that specific column, by adding a single subscript to the 2D array variable name.

Here's the basic syntax:

  • Define a 2D array variable, grid: int[][] grid;
  • Instantiate the array with 3 rows, 4 columns: grid = new int[3][4];
  • Find the number of rows in the array: int rows = grid.length;
  • Find the number of columns (in first row): int cols = grid[0].length;
  • Access first element in first row: int first = grid[0][0];
  • Access last element in last row (assume that rows and cols already defined.)
    int last = grid[rows - 1][cols - 1];

2D Arrays and Loops

Normally, you'll use a nested loop to process the elements in a 2D array. The basic skeleton for this loop, (which works for all rectangular 2D arrays), looks like this:

final int ROWS = grid.length; // get the rows
final int COLS = grid[0].length; // get the rows
for (int row = 0; row < ROWS; row++)
    for (int col = 0; col < COLS; col++)
        // process grid[row][col] here

2D Arrays and Methods

With a 2D array, you can write methods that process the entire array, or methods that process a single row. Here are the most common kinds of methods, (using a double array named prices):

  • Process the entire array (using a nested loop): double averageAll(double[][] prices) {}
  • Process one row in the array: double averageRow(double[] row) { }.
    Call the method (using prices) like this: double ra = averageRow(prices[0]);
  • To process an array column, you must pass the array and a column number:
    double averageCol(double[] prices, double colNum) { }
