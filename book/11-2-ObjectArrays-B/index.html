Initializing the Elements

If the deck variable were an array of double or int, each element would automatically be initialized to 0, so you could use the array immediately. With object arrays, however, each element is initialized to the special value null like this:
Notice in this picture:

  • When we create the array variable deck that the variable contains null, since we haven't created any GImage objects yet.
  • Next, when we instantiate the array, Java creates 52 GImage object references (although this picture only shows the first five), and stores a reference that points to those array elements inside the array variable deck

Note, though, that the array elements themselves are not GImage objects, but GImage references. And, just like the original array variable deck, each of those references contains the value null.

The value null is not very useful in its own right. Actually, it's not useful at all. Before you can use deck you have to fill each element with a valid GImage reference by using the new operator on each element in the array like this:

int len = deck.length;
for (int i=0; i < len; i++)
{
    // Calculate the filename for this image
    deck[i] = new GImage(imageName);
}

Once you're finished, each element in the deck array points to a new GImage card image, as this illustration shows:
As you can see, every object array requires three steps to be useful:

  1. Declare an array variable.
  2. Create (or instantiate) the array object.
  3. Create and attach objects for each element in the array.

Object Arrays Summary

  • When an array contains objects you must separately instantiate the array elements. When the array is instantiated, each element is initialized to a null reference.
  • You can instantiate the elements in an object array by using a loop or by using an initializer list.
  • When an array contains objects, use the subscripted element name as the receiver when you call any of the object methods: pixels[i].setRed(0).
