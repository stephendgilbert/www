CS 170—Week 9

Exercises & Assignments


This week you should make sure that you can write programs using arrays, methods and loops. The problems below go from simple (array access with no loops), to complex. Here's the list of things you should do this week.

  1. Arrays and Methods—ArrayMethodProblems: Complete the following five additional practice problems in the ArrayMethodProblems class. These problems do not require you to use loops. Instead, they involve simple array access.
    • commonEnd
    • reverse3
    • rotateLeft3
    • makeLast
    • midThree

    You can check your answers with the Check program, and look at the solutions if you get stuck.

  2. Arrays and Loops—ArrayLoopProblems: Complete these extra array loop practice problems in the ArrayLoopProblems class.
    • arrayCount9
    • arrayFront9
    • array667
    • noTriples
    • has271

    You can check your answers with the Check program, and look at the solutions if you get stuck.

  3. Buttons & Text Fields—BillBoard: Create an ACM Graphics program named BillBoard that displays a message. Your program will have widgets to allow the user to enter a new message and to change the color of the display. Here's what it looks like as it runs:
    Here are the instructions:
    • Making the program 600 wide and 200 high. Center the message vertically on the screen. To set the size for an ACM graphical application, just define these two instance variables as part of your class:
      public static final int APPLICATION_WIDTH = 600;
      public static final int APPLICATION_HEIGHT = 200;
    • Use a GLabel (not a JLabel) for the message. You'll also need two JButtons and a JTextField as shown in the example here.
    • When you position your label, center it horizontally.
    • When the Show It! button is pressed, grab the text from the JTextField and use it to change the message displayed in your label. When you do this, you need to re-center the GLabel Note: you should not create a new GLabel object when you do this. Just call the object's methods to change its location and contents.
    • When the Color button is pressed, select a new color. You can use an array of colors and cycle through them, or you can choose a color at random. Remember that the ACM RandomGenerator class has a nextColor() method that will do this for you.

    There is no Check program for this assignment but I've posted a solution on the Solutions page.

  4. Events & Animation—Marquee: Modify the BillBoard program to make the message move as it is displayed:

    Here are the instructions:

    • Use the instructions in Section 8.4—Introducing Animation to add a run() method (with the standard animation loop), and an advanceOneFrame() method.
    • When you first position your label, position it off the right side of the screen. As you draw each frame of your animation, move it to the left by two pixels. When the entire label has scrolled off the screen to the left, position it back on the right-hand side again.
    • Each time the advanceOneFrame() method is called, simply change the position of the label. Do not create a new label object.

    There is no Check program for this assignment but I've posted a solution on the Solutions page.
