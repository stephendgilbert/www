CS 170—Chapter 14

Exercises & Assignments


In this chapter you should get some practice writing classes with inheritance and composition, along with a chance to work with some ArrayList problems. Here's the list of things you should do:

  1. ACM Inheritance—GBall: Read Section 14.3—Inheritance at Work and complete the GBall class. This will give you some practical experience extending the ACM graphical classes.

    As you develop your class, you'll use the PolarBounce2 program to exercise it. To submit your finished GBall class, though, use the Check program, and the unit test that I've supplied.

  2. Widgets and Iheritance—Dot: You don't need the ACM classes to do inheritance; you can create new versions of the standard Java widget classes just as easily. To see how, read the bottom section of Section 14.3—Inheritance at Work and complete the Dot class, which extends the standard Component class from Java's java.awt package.

    Create the TestDots test harness as you read along with the lecture. To submit your assignment, though, use the Check program.


  3. Applying Composition—GBall2: While inheritance allows you to extend existing classes, providing new capabilities, or overriding existing features, composition allows you to combine the capabilities of simple unrelated classes in a new, more complex class.

    To see how, read Section 14.4—Composite Objects and complete the GBall2 class, which extends the GCompound class, but contains a GOval object. You should compare this version with the original GBall, which extended GOval.

    For this assigment, you'll use the PolarBounce4 program to exercise it. To submit your finished GBall2 class, though, use the Check program, and the unit test that I've supplied.


  4. Inheritance—the Triangle Class: The Chapter14 code folder contains a class named GeometricObject. Without modifying this class in any way, extend it to create a new class named Triangle. The Triangle class contains:
    • a single field named sides that consists of an array of three double values, representing side1, side2, and side3. You must use an array; do not use individual instance variables. Each element should have default values of 1.0 to represent the three sides of the triangle.
    • a no-arg constructor that creates a default Triangle: (new Triangle()).
    • a constructor that creates a Triangle with the three sides specified: (new Triangle(1.0, 2.0, 1.5)).
    • a third constructor that takes the three sides, the Color and whether or not the object should be filled: (new Triangle(1.0, 2.0, 1.5, Color.RED, true)).
    • A method named getArea() that returns the area of the Triangle. (Look up the formula online.)
    • A method named getPerimeter().

      Note that your Triangle class will only supply the portion that begins with the word Triangle. Use the inherited toString() method to generate the first portion of the returned value.

    To submit your finished Triangle class, use the Check program, and the unit test that I've supplied.

  5. Working with ArrayList: Try your hand at two more difficult ArrayList processing functions, the copyEvens() and the copyEndy() problems.
