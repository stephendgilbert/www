CS 170—Week 6

Exercises and Assignments


The big idea this week is using the if statement to make decisions, using the logical operators to combine decisions, and writing decisions that have more than one output. You'll use those two when writing your own functions. Here's the list of things you should do this week.

  1. Writing Simple Logic Functions: LogicProblems: As you did last week with the String functions, you should try to complete the remaining String functions in LogicProblems.java. To get started, walk through the selection section at the beginning of Section 6.5—Flow-of-Control Functions. That will get you started with the first exercise. Once you're finished, complete the problems:
    • sleepIn
    • sumDouble
    • diff21
    • makes10
    • nearHundred

    Even if you already know how to use nested if statements, for this assignment, try to solve these using only simple if statements along with the return statement. Don't use any of the binary Boolean operators like && or ||. You may use the Boolean not operator (!), even though we haven't covered it yet.

    Just like last week, you can work on these problems without using DrJava by working with the online homework site CodingBat.com where you'll find them in the Warmup-1 section. That's what the solution videos here use. Remember, though, if you do use CodingBat instead of DrJava and Check, you cannot use the static modifier on your functions. With DrJava and Check, you may (so your code will look like that shown in your text), but you don't need to.

  2. Combining Decisions—LogicalOperators: You can get some extra practice with the logical operators by completing these predicate functions in LogicalOperators.java. To get started, walk through the hands-on exercise in Section 6.6—Hands-On Selection. That will get you started with the first exercise. Once you're finished, you can continue with these problems:
    • in1To10
    • redTicket
    • sortaSum
    • withoutDoubles
    • teaParty

    Complete these problems using if statements along with the binary Boolean operators like && and ||, along with the not operator (!).

    You can check your answers with the Check program, and look at the solutions if you get stuck.

  3. Multi-way Branches—MultipleChoice: This is the third set of decision functions this week. Like the other examples in Section 6.6—Hands-On Selection, these require you to write functions using sequential or nested if statements. Do not use the logical operators.
    • evenlySpaced
    • luckySum
    • loneSum
    • makeBricks
    • makeChocolate

    You can check your answers with the Check program, and look at the solutions if you get stuck.
