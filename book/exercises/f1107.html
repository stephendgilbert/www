CS 170—Week 7

Exercises & Assignments


This week you learned how to repeat a series of actions, using both the for loop and the while loop. These extra exercises will give you some more practice writing functions that use these skills. Here's the list of things you should do this week.

  1. String Functions With Loops: LoopProblems: This is the second set of functions this week. Like the first two examples in Section 7.4—Loop Functions, these require you to write String functions using the for loop to solve each problem.
    • stringTimes
    • stringE
    • catDog
    • countCode
    • bobThere

    If you work at CodingBat.com you'll find these problems in the String-2 section. If you use DrJava, then you'll find them in the LoopProblems.java file in this chapter's code folder. You can check your answers with Check, and look at the solutions if you get stuck.

  2. The while Loop—WhileLoopsProblems: This set of functions this week all involve more complex processing. Like the problem at the end of Section 7.4—Loop Functions, these require you to write functions using the while loop, even though many of them could be solved easier using the for loop.
    • repeatSeparator
    • xyzMiddle
    • sameStarChar
    • starOut
    • wordEnds

    You can check your answers with the Check program, and look at the solutions if you get stuck.

  3. ACM Graphical Shapes—GraphicsHierarchy: Write an ACM GraphicsProgram, named GraphicsHierarchy that draws the partial diagram of the acm.graphics class hierarchy as shown here:
    The only classes you need to create this picture are GRect, GLabel, and GLine. The tricky part is specifying the coordinates so that the different elements of the picture are aligned properly. The aspects of the alignment for which you are responsible are:
    • The width and height of the class boxes should be specified as named constants so that they are easy to change.
    • The labels should be centered in their boxes. You can find the width of a label by calling label.getWidth() and the height it extends above the baseline by calling label.getAscent(). If you want to center a label, you need to shift its origin by half of these distances in each direction.
    • The connecting lines should start and end at the center of the appropriate edge of the box.
    • The entire figure should be centered in the window.

    Define the public static final int values APPLICATION_WIDTH and APPLICATION_HEIGHT to specify the width and height of the application when it runs. The size should be 600 by 300.

    There is no Check program for this exercise, but you may look at the solution page.

  4. Images and Playing Cards—FiveCards: Using the TwoCards example from Chapter 7 as a model, write write a program named FiveCards that deals five cards face up on a table.
    • Display them nicely either side-by-side or slightly overlapped.
    • The cards are available inside the cards folder in this chapter's code folder.
    • Use the RandomGenerator() class to select random cards to deal.

    Picking a Random Card: Create a single String variable containing the names of all of the card images like: "2c3c4c...". Then, generate a random number and then use the random number to pick the element to display, by using the substring() method. If cards is the name of your String variable, and cardNo is a random integer between 0 and 52, you can pick a random card with cards.substring(cardNo * 2, cardNo * 2 + 2).

    Note that the program is not interactive. All cards can be dealt face up. They don't have to flip over like you did with TwoCards.

    There is no Check program for this assignment, but I should have something on the solutions page.
