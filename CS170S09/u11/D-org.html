Lesson

This lesson will teach you how to create and use arrays that contain objects instead of those that contain primitive values. Instead of using individual Button objects in your applets, you may want to use arrays of Buttons, like those pictured in the applet here.

In addition, you'll learn to work two array-like classes, Vector and ArrayList, that are especially designed to hold objects. Unlike the SimpleArrayClass that you built in the last unit, which could store only a single type, both of these classes can hold any type of object.

And, like the ResizableArrayClass that you also worked on, both Vector and ArrayList will automatically resize themselves as new elements are added, unlike the built-in arrays.

Let's start, though, by looking at storing objects in POA (Plain Old Arrays).

Arrays Of Objects

You create an array of objects using exactly the same syntax you use to create an array of primitive types. Here, for instance, is how you create an array of Button objects for the toolbar that appears at the top of the applet pictured at the beginning of this lesson:

private Button[] toolBar = new Button[5];

Up to this point, objects and primitive arrays work just the same. Now, however, the two diverge.

The Vector Class

One of the problems with using arrays, is that they are fixed-size. If you plan for ten elements when you create your array, you can't change your mind after the program is running, and add an eleventh element. A Vector object, however, lets you do just that.

You probably know the difference between bitmap and vector graphics, have heard the term disease vector related to a potential bird flu epidemic, or referred to wind vectors while sailing, or in a physics class. You might even be familiar with the video-game character Vector, from Sonic the Hedgehog, shown here.

What, you might wonder, do all of these have in common with Java's Vector class? Good question.
The answer is: not a thing!

The Vector class implements an "automatic" array, similar to the one that you created at the end of the last lesson. Like the class that you wrote, when you add an eleventh element to a ten-element Vector, the Vector will automatically resize itself.

One difference between your class, which stored primitive values, and the Vector class, is that Vectors can only hold objects; if you want to store primitive types in a Vector, you have to "wrap" each primitive value up in one of Java's wrapper classes (Integer, Double, etc.).

The term vector is used in the physics world to mean an object that has both a magnitude and a direction; this physics definition is the one used with vector graphics, disease propogation, and weather analysis. Mathematicians, though, also use the word vector to refer to an array, and it is this variant of the term that the Java programmers adopted, (along with the authors of the C++ Standard Template Library), as the name of the original array class.

Using Vectors

Let's look at the various steps you follow to use a Vector. Then, when you're finished, we'll revisit ToolBar.java, using a Vector to make it a little more flexible.

The Vector class is in the java.util package, so you have to remember to add:

import java.util.*;

to any programs that use Vectors, just as when using the Date or Random classes.

Since Vector is a class, and not a built-in language feature like arrays, you use a regular constructor to create a Vector object. Here are three examples:

Vector v1 = new Vector();
Vector v2 = new Vector(100);
Vector v3 = new Vector(100, 25);

Capacity

When you create a Vector using the default constructor, like the Vector v1 in this example, it can initially hold 10 elements. This is called the Vector's capacity. If you like, you can supply an initial capacity as Vector v2 does.

Once you've added ten elements to v1, or one-hundred elements to v2, adding an additional element causes the Vector to grow. Both v1 and v2 will grow by doubling their capacity. After adding an 11th element to v1, its new capacity will be 20. After adding a 101th element to v2, its new capacity will be 200.

If this strategy of doubling each time a Vector grows causes problems you may specify a third constructor, used for the Vector v3. This constructor allows you to supply both an initial capacity as well as a growth factor. Each time the Vector v3 grows, it will simply increase by 25 elements, no matter how large it gets.

Adding Elements

You can add objects to a Vector using one of two methods. Here's an example that shows both methods using two Button objects, named one and zero.

Button one = new Button("One");
Button zero = new Button("Zero");
v1.addElement(one);
v1.insertElementAt(zero, 0);

addElement() and add()

The addElement() method adds the new element to the end of the Vector. Vectors are indexed, just like arrays, starting at zero. After adding the Button one to the Vector v1, the Button one is in position 0.

In Java 2, the add() method, which is part of the List interface, was retrofitted to the Vector class. Using add() instead of addElement() allows your Vector to interact with other Java 2 collection classes.

insertElementAt() and add()

The second method for adding elements to a Vector, insertElementAt(), allows you to specify which position the new element should occupy. When you use insertElementAt() the new element does not replace what was previously at that location. Instead, all of the elements in the Vector are shifted "up" one to make room for the new arrival.

After inserting the Button zero at position 0, the Button one is moved to position 1.

In Java 2, you can use the add(Object, int) method to do the same thing. As with the simpler version of the add() method, this allows your Vector objects to interact with other Java 2 collection classes.

Using Vector Elements

Arrays are easy to use because you can use simple assignment to replace or retrieve the value stored in a particular element. You can't do that with the Vector class. Instead, you have to use one method to replace the current value in an element:

setElementAt(Object o, int index); // Java 1.1 on
set(int index, Object o); // Java 2 only

and another to retrieve the element:

elementAt(int index); // Java 1.1 on
get(int index); // Java 2 only

Note that the Java 2-only code is, again, simpler than the original version. Note also that the setElementAt() and set() methods use a different order for their arguments. The revised methods are designed to "correspond to" the same order as assignment, and read a little more naturally.

Also, because Vectors really hold Object references, and not Button or Label references, you have to cast the Object reference returned from elementAt() to the correct "real" type before you use it, like this:

Button first = (Button) (toolBarV.elementAt(0));
// This won't work ->> Button first = toolBarV.elementAt(0);

Although storing and retrieving objects in a Vector is more work than using an array, many common operations that you have to perform "by hand" when using arrays are automatic when using Vectors.

For instance:

  • To find out if a particular object is contained in a Vector, you can use the contains() method.
  • If you already know that a particular object is located in a Vector, you can retrieve its position using indexOf().
  • You can remove an object from a Vector using removeElementAt(). When you do this, the other objects move "down" to close up the gap left by the departee.

When you wrote your own array classes, these were all things that you had to do on your own by writing various loops.

The ArrayList Class

There are several reasons that people might prefer an array to a class like Vector: Vectors are a little less convenient (since you can't use subscript notation), they are generally less efficient (since you can't store primitive values).

Vectors are also intrinsically "unsafe", since they don't keep track of what kind of elements they actually store. As far as the Vector class is concerned, everything is an Object.

The new Java 5 collection classes change this. In Java 5, you can specify the type of element stored in a collection, by using a type parameter, a facility that is known as generics. The collection class that's most similar to the Vector class is called the ArrayList.

Remember, if you compile your code using the new generic classes, your code won't run on pre-Java 5 Virtual Machines. This is a Java 5 only feature.
