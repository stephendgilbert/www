Operator Overloading

  • Convenient shorthand
  • More intuitive: a + b * c vs. plus(a, times(b, c))
  • Same technique can make code confusing (apply the / operator to strings)
  • C++ supports a rich set of operators
  • If the intended meaning of an operator is not immediately clear (to, e.g., your grandmother), then don't overload it

Operator Overloading

  • Programmer may define his/her own, as a
    • Simple (non-member) function
    • Member function
  • Powerful and subtle feature of C++
  • Advantages:
    • Easier to remember
    • Reuse of existing code
    • Concise description of the task
  • Caveat:
    • Lack of context may disguise meaning

Operator Overloading

To define an operator:

  • One argument (operand) must be a class type
  • I.e., can't redefine existing behavior
  • Can't change precedence, associativity, or arity
  • Can't invent new symbols
  • Not all operators may be overloaded

Operator Member Functions

  • First interpreted as a member function of the left-most operand
  • Subsequent arguments are passed to the member function: Time later = now + 60;

    becomes

    Time later = now.operator+(60);
  • So, a binary operator has only one argument

Operator Overloading (member vs. global)

  • Some operators (e.g., assignment) are required to be members

When choosing, keep in mind:

  1. Non-members have only public access
    • Or, make it a friend (see Chapter 18)
  2. No implicit conversion for implicit parameters

Member function is preferable

  • if the left argument is modified (+=)
  • if the data fields are not easily accessible

Global function is preferable

  • to allow type conversions on the left operand

GCD — Greatest Common Divisor

  • By Euclid, 3rd century BCE
  • In The Elements
  • (Considered to be) among the first formally developed algorithms
  • If d is the gcd of m and n (assume m < n) :
    • d divides m and n
    • n/d and m/d are integers, so
    • n/d − m/d is an integer, so
    • (n − m)/d is an integer, which means
    • d divides (n − m )

Overloading Simple Arithmetic Operators

  • = + - * / %
  • Can be written as a two-argument global function, or a one-argument member function Fraction operator+(const Fraction& left, const Fraction& right) { Fraction result(left.numerator() * right.denominator() + right.numerator() * left.denominator(), left.denominator() * right.denominator()); return result; }
  • Remaining operations defined similarly
  • Supply behavior that is analogous to the arithmetic operators; intuitive
  • Parameter types need not match (Time::operator+)

Overloading Unary Arithmetic Operators

  • + - * & have both binary and unary forms
  • To overload unary form, reduce the number of parameters by one
  • E.g.: Fraction operator-(const Fraction& value) { Fraction result(-value.numerator(), value.denominator()); return result; }
  • A unary operator defined as a member function takes no arguments (example in Section 14.6)

Overloading Comparison Operators

  • Similar to the arithmetic operators
  • Not supplied automatically
  • Typically return a bool
  • E.g.: Times are equal <=> difference is 0 bool operator==(const Time& a, const Time& b) { return a.seconds_from(b) == 0; }
  • Less-than for Fractions: bool operator<(const Fraction& left, const Fraction& right) { return numerator() * right.denominator() < denominator() * right.numerator(); }

Productivity Hint

Define Comparisons in Terms of Each Other

  • Write 1 or 2 member functions; equality, and strict inequality (less than)
  • Or, a single compare() method:
    • Returns 0 if args are equal
    • Negative if left arg is less than the right arg
    • Positive otherwise
    int Fraction::compare(const Fraction& right) const { return numerator() * right.denominator() - denominator * right.numerator(); // Return the numerator of the difference }

Define Comparisons in Terms of Each Other

  • Define all comparisons in terms of these functions
bool operator<(const Fraction& left, const Fraction& right) { return left.compare(right) < 0; } bool operator==(const Fraction& left, const Fraction& right) { return left.compare(right) == 0; } bool operator<=(const Fraction& left, const Fraction& right) { return left.compare(right) <= 0; }

Symmetry and Conversion

  • Arithmetic and comparison operators are generally nonmember functions
  • No implicit conversions of left operand for member functions:

    Consider Fraction a;

    • (a == 2) works fine (given the constructors)
    • (2 == a) will not work if equality is a member function.
  • These both work fine if == is a nonmember function

Overloading Input and Output

<< and >> operators

  • Bit-shift operators (if operands are integers types)
  • In C++, commonly used to define stream insertion and extraction operations
  • Overloaded just like the binary arithmetic operators
  • Should be non-member functions

Stream Output

  • ostream class allows a character to be written to a stream
  • << already overloaded for all the built-in types: ostream& operator<<(ostream& out, char value); ostream& operator<<(ostream& out, int value); ostream& operator<<(ostream& out, float value); ostream& operator<<(ostream& out, double value); ostream& operator<<(ostream& out, bool value); ostream& operator<<(ostream& out, const char* value); ostream& operator<<(ostream& out, string value);
  • Left operand is a stream
  • Right operand is some value to be written

Stream Output <<

  • << associates left-to-right
  • The stream itself is returned
  • This allows more complex expressions: int i = 34; cout << i << "\n";

Stream Output <<

  • E.g.: Overload << for Fraction ostream& operator<<(ostream& out, const Fraction& value) { out << value.numerator() << "/" << value.denominator(); return out; }
  • So we can now: Fraction n(3, 4); cout << "The value of n is " << n << "\n";

Stream Input

  • Stream library provides for all built-in types
  • Can be overloaded, much like the output operator
  • 2nd parameter is a non-const reference
  • E.g.: Input operator for Time
    Assume Time is input as 3 separate integers: 9 15 00 to represent 9:15:00 a.m. istream& operator>>(istream& in, Time& a) { int hours; int minutes; int seconds; in >> hours >> minutes >> seconds; a = Time(hours, minutes, seconds); return in; }

Peeking at the Input

You may put a single character back into certain streams

E.g.: We want to allow a user to input a Fraction as a single integer (7), or as an integer, slash, integer (3/4)

  • Read numerator
  • If next character is /, discard it, read denominator

Overloading Increment and Decrement Operators

  • Typically alter the operand (argument)
  • Generally defined as member functions
  • Two versions of each:
    1. prefix version:  ++x
    2. postfix version:  x++
  • C++ uses an extra argument to distinguish the postfix form class Fraction { ... Fraction& operator++();  // Prefix form Fraction operator++(int unused);  // Postfix form ... };

Overloading Increment and Decrement Operators

Fraction& Fraction::operator++() { top += bottom; return *this; } Fraction Fraction::operator++(int unused) { Fraction clone(top, bottom); top += bottom; return clone; }
  • Prefix returns a reference
  • Postfix does more work

Quality Tip

Avoid Dependencies on Order of Evaluation

  • ++x — increments x, then evaluates
  • x++ — evaluates x, then increments
  • Statements like these are a common source of errors: int i = 0; int j = 0; vector<double> s(10); double a = s[i++]; // a is s[0], i is 1 double b = s[++j]; // b is s[1], j is 1

Quality Tip

Avoid Dependencies on Order of Evaluation

  • C++ does not define the order that operands are evaluated for most operators
  • This code is not only confusing, it is ambiguous: int i = 5; int x = i + ++i;
  • x is either 10 or 11
  • The compiler will do whatever it feels is most efficient
  • Subtle error; it may appear to work, but it is unreliable

Overloading the Assignment Operators

  • Must be a member function
  • Is automatically generated
    • Member-wise copy (shallow copy)
    • Default and copy constructors and destructor also supplied by compiler
  • Default behavior sufficient for many classes (including our Fraction class)
  • Explicitly supply if class has outside references (see Chapter 15)

Overloading Compound Assignment Operators

+= -= *= , etc., combine operations of arithmetic and assignment

  • Never supplied automatically
  • If, e.g., + is defined, and assignment makes sense, then += should be supplied Fraction& Fraction::operator+=(const Fraction& right) { top = top * right.denominator() + bottom * right.numerator(); bottom *= right.denominator(); normalize(); return *this; }

Define One Operator in Terms of Another

  • Ensures consistent behavior
  • Given Fraction::operator+=, define +: Fraction operator+(const Fraction& left, const Fraction& right) { Fraction clone(left); // Copy the left argument clone += right; // Add the right argument to it return clone; // Return the updated value }

Overloading Conversion Operators

  • C++ performs a wide variety of implicit conversions (coercions)
  • E.g., float multiplication is performed here: 6 * 3.1415926
  • For user-defined classes, there are 2 categories of conversions to consider:
    From a type to the new user-defined type
    Accomplished with constructors
    From the new user-defined type to another type
    Uses a conversion operator

Overloading Conversion Operators

Constructors

  • Conversion to a user-defined type is handled by a constructor
  • E.g., Fraction has a constructor that takes a single integer
  • Addition is defined on 2 Fractions
  • C++ will use this to convert an integer to a Fraction
  • So, addition between a Fraction and an integer will be performed
  • Note: The compiler performs at most one level of user-defined type conversion when matching an overloaded function

Overloading Conversion Operators

  • To add: Fraction a(3, 4); a = a + 2;  // Addition of Fraction and integer
  • Two choices will be explored:
    • Convert Fraction to int ? Can't
    • Can convert an int to a Fraction
  • Uses the constructor to convert the integer into an unnamed Fraction object
  • Then does normal Fraction addition a = a + Fraction(2);

Overloading Conversion Operators

  • If addition is not a member function, this works too: b = 2 + a;
  • Use explicit keyword to inhibit implicit conversions (coercions) class Fraction { explicit Fraction( int ); ... }

Overloading Conversion Operators

Conversion Operators

  • Convert from a user-defined type
  • An operator must be supplied for each type to convert to
  • Definition:
    • Must be a member function
    • Has a type as its name
    • Takes no arguments
    • Has no return type
  • May be used implicitly, or called explicitly (cast)

Overloading the Function Call Operator

  • Objects can be used as functions (function object)
  • Must be defined as a member function
  • Can carry (and change) its own data values
  • Only operator where the number of arguments is not fixed
  • Can be passed where template functions can't
  • Used extensively by various generic algorithms in the STL

The Function Call Operator

  • Simple to write a function that takes no arguments and returns a random number on [1, 100] int rand_100() { return 1 + rand() % 100; }
  • More difficult to write a similar function (no args) that returns a value on [a, b], where a and b aren't known until run time
  • A function object makes it simple:

The Function Call Operator — RandomInt

class RandomInt { public: RandomInt(int ia, int ib); int operator()(); private: int a, b; }; RandomInt::RandomInt(int ia, int ib) : a(ia), b(ib) {} int RandomInt::operator()() { return a + rand() % (b - a + 1); }

The Function Call Operator

Declare an instance, then use it as a function:

RandomInt a(7, 15); // Return random values from 7 to 15 cout << "one random value " << a() << "\n"; cout << "and another " << a() << "\n";

Inline Functions

  • Avoid overhead of function calls
    • pushing arguments on to the activation record stack, transfer of control, return from function, popping arguments off the stack
  • Body expanded in place of each call
    • So, the program gets larger
  • Only for very short functions:
    • three or fewer assignment statements,
    • a single conditional (if) statement, and/or
    • a return statement
  • Compiler will inline what it wants to
