The if statement

  • Used to implement a decision
  • Has two parts: a condition and a body
  • Example: 
    if (area < 0)
   cout << "Error: Negative area\n";
  • Body is true only if condition is true

Figure 1 A Decision

The if statement

Body may have multiple statements: 

if (area < 0)
{
   cout << "Error: Negative area\n";
   length = 0;
}
  • Must be enclosed in curly braces {}
    • Called a block statement
  • Executed sequentially
  • Curly braces may be used with a single statement
    • It's recommended

The if/else statement

  • To perform an action if test is true, and a different action in all other cases
  • Add an else followed by a statement, or a block: 
    if (area >= 0)
   cout << "The side length is " << sqrt(area) << "\n";
else
   cout << "Error: Negative area\n";

Figure 2 if/else Statement

Syntax : if Statement

if (condition) statement1 if (condition) statement1 else statement2

Example: 

if (x >= 0) y = sqrt(x);
if (x >= 0) y = sqrt(x); else {y = 0; cout << "Error";}

Purpose:

Execute a statement if the condition is true. When paired with else, execute the second statement if the condition is false. The statement can be a simple statement (ending in a semicolon) or a block statement.

Syntax : Block Statement

{ statement1 statement2 ... statementn }

Example: 

{
   double length = sqrt(area);
   cout << area << "\n";
}

Purpose:

Group several statements into a block that can be controlled by another statement.

The Selection Operator

  • C++'s only ternary operator (cannot be overloaded): 
     test ? value1 : value2
  • It is an expression (has a value)
  • E.g., 
    y = x >= 0 ? x : -x;
    is equivalent to 
    if (x >= 0) y = x;
else y = -x;

Relational Operators

  • Used to compare numbers and strings

Relational Operators

  • Return a boolean (true or false)
  • Note: = is used for assignment
  • == tests for equality
  • In C++, any zero value is false; all others are true
  • Don't do this: 
    if (x  = 7) ...
    • Will always be true
    • x will have the value 7 afterwards

Relational Operators (strings)

  • These six operators work on instances of the string class
  • Do not work (as intended) for C-strings (char*)!
  • Computed using dictionary order (in a way). This test: 
    string name = "Tom";
if (name < "Dick") ...
    fails
  • But, in ASCII, upper-case precede lower-case, so 
    if (name < "dick")
    is true
  • Just as in the dictionary, leading substrings appear first: 
    if (name < "Tomorrow")
    is true

Quality Tip

Compile with Zero Warnings

  • Compiler emits 2 messages: errors and warnings
  • Errors are fatal; no executable is produced
  • Warnings are not fatal
  • Do not accept a warning, unless you can explain and justify it
  • Warnings usually indicate that your thinking isn't quite correct, and may well cause your program to misbehave
  • Endeavor to write code that emits no warnings

Comparison of Floating-Point Numbers

  • Float types have limited precision
  • Rounding errors are unavoidable: 
    sqrt(2) * sqrt(2) == 2
    is false.
    • sqrt(2) * sqrt(2) is 2.0000000000000004
    • The default display won't show this; it'll look like 2
  • To compare float-types x and y:
    |x - y| ≤ ε
  • Choose ε based on magnitude of x and y

The switch Statement (cont.)

  • Test cases must be constant integer-types

Common Error

The Dangling else Problem

  • else is matched to the closest preceding if
  • Consider this code: 
    double shipping_charge = 5.00; // $5 inside continental U.S.

if (country == "USA")
   if (state == "HI")
      shipping_charge = 10.00; // Hawaii is more expensive
else // Pitfall!
   shipping_charge = 20.00; // as are foreign shipments
  • Indentation suggests that the else is paired with the test country == "USA"
  • Compiler ignores white space

Common Error

The Dangling else Problem (cont.)

  • Use an empty else: 
    if (country == "USA")
   if (state == "HI")
      shipping_charge = 10.00; // Hawaii is more expensive
   else;
else 
   shipping_charge = 20.00; // as are foreign shipments
  • Or, use braces: 
    if (country == "USA")
{
   if (state == "HI")
      shipping_charge = 10.00; // Hawaii is more expensive
}
else 
   shipping_charge = 20.00; // as are foreign shipments
  • Braces are always recommended, to avoid this an other problems

Boolean Operations

  • Boolean operations (after George Boole) have Boolean (true/false) expressions as operands
  • Modify or combine conditions into larger, more complex Boolean expressions
  • Two binary operations in C++:
    &&
    Logical and
    Is true when both operands are true
    ||
    Logical or
    Is true when either operand is true

Boolean Operations (examples)

  • Following test is true only when both country is "USA", and the weight is < 1 
    if (country == "USA" && weight <= 1)
   shipping_charge = 2.50;
  • This test is true if state is either "HI" or "AK": 
    if (state == "HI" || state == "AK")
   shipping_charge = 10.00;

Boolean Operations (bool type)

  • C++ has a Boolean type: bool
  • Can be true or false
  • Useful for storing a value to be used later: 
    bool discounted = country == "USA" && weight <= 1;
...
if (discounted)
   ...
  • Also makes the code more legible
  • C doesn't have a bool type; we used any integer type (char, short, int, etc)
  • Note, for compatibility reasons, in C++ any nonzero value evaluates to True
  • Be careful

Boolean Operations (not operator)

  • The ! operator (said "not") inverts the value of any Boolean expression 
    if (! discounted)
  • Unary operator

Boolean Operations (lazy evaluation)

These two operators are evaluated lazily

  • Operands are evaluated left-to-right
  • Evaluation ends as soon as the truth value is determined
  • Given X is true and Y is false: 
     X || Y
    is true; Y is never evaluated 
     Y && X
    is false; X is never evaluated
  • This is quite handy: 
     (n > 0) && (tot/n < 0.9)

Common Error

Multiple Relational Operators 

if (-0.5 <= x <= 0.5) // Error
  • Makes sense in mathematics
  • Does not make sense in C++
  • -0.5 <= x evaluates to true or false
  • You would then compare that Boolean value to an integer
  • Needs to be done this way: 
    if (-0.5 <= x && x <= 0.5) ...

Advanced Topic

De Morgan's Law

!(A && B) ≡ !A || !B
!(A || B) ≡ !A && !B

  • Use these to simplify expressions: 
    if (!(country == "USA"
      && state != "AK"
      && state != "HI"))
   shipping_charge = 20.00;
    becomes 
    if (country != "USA"
      || state == "AK"
      || state == "HI")
   shipping_charge = 20.00;

Syntax : for Statement 

for (initialization_statement; condition; update_expression) statement

Example: 

for (int i = 1; i <= 10; i++) sum = sum + i;

Purpose:

Repeatedly execute a statement, typically while updating a variable in regular increments. The initialization statement is executed once. While the condition remains true, the statement and the update expression are executed.

Quality Tip

Don't use != to Test the End of a Numeric Range

  • Hidden danger: 
    for (i = 1; i != nyear; i++)
{
   ...
}
  • If nyear is negative?
  • Always use the strongest test: 
    for (i = 0; i < nyear; i++)
  • Remember, never test float types for equality

Common Error

Forgetting a Semicolon — Empty Loop Body

  • Occasionally, all the work is done in the loop header: 
    string filename; // e.g., hello.cpp
string name;
...
for (i = 0; filename.substr(i, 1) != "."; i++)
   ;
name = filename.substr(0, i); // e.g., hello
  • Consider the effect of forgetting the first semicolon (actually, not disastrous in this example)
  • Write this as a while loop: 
    int i = 0;
while (filename.substr(i, 1) != ".")
   i++;

  • A special value used to signal termination is called a sentinel
  • Only use a value as a sentinel if it's not valid input
  • Do not use arbitrary, but valid, values (999, 9999, etc)

Processing Inputs (input failures)

  • If the input stream fails (e.g., entering a non-numeric string when an integer is expected), it enters a fail state
  • To test: 
    if (cin.fail())
  • The EOF (end of file) will also set the fail state
  • To signal EOF, use ctrl-D (on Unix and Mac), or ctrl-Z (on Windows)

Processing Inputs (input failures)

  • If the object itself is evaluated as a Boolean, it will return its state (True means that the stream is okay): 
    if (! cin)
  • The >> operator returns the stream object (LHS)
  • This is a common idiom: 
    cout << "Enter values, Q to quit.\n";
while (cin >> values)
{
   sum = sum + input;
   count++;
}

Productivity Hint

Redirection of Input and Output

  • Values (inputs) are often stored in a file
  • The shell (command line interface) on your system allows you to redirect standard input, so that it is read from a file, rather than from the keyboard: 
    average < numbers.txt
  • Similarly, output can be redirected to a disk file, rather than the screen (console): 
    average < numbers.txt > output.txt

Clearing the Failure State of a Stream

  • Once the fail bit is set, all further operations on that stream also fail
  • Call the clear() method:

Simulation (ranges of random floats)

  • Dropping a needle has many possible outcomes
  • Position and orientation are not discrete values
  • To generate a floating-point number on [a, b]: 
    double r = a + (b - a) * (rand() * 1.0 / RAND_MAX);
