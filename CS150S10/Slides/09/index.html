Intro

  • C++ I/O library based on streams
  • input stream — source of data
  • output stream — destination for data
  • Wrappers (inherited classes) provide stream access to files (including the screen and keyboard), strings, etc.

Reading and Writing Text Files

  • Include the header file fstream
  • Use a file variable to access a disk file:
    ifstream
    For reading (input)
    ofstream
    For writing (output)
    fstream
    For both reading and writing
  • For example: ifstream input_data;

Reading and Writing Text Files

Input

  • File must be opened:input_data.open("input.dat");
  • Associates a disk file with a file variable
  • input_data is a stream
  • Can be used much like cin: int n; double x; input_data >> n >> x;
  • Read strings in the familiar way: string s; input_data >> s; // Read a word getline(input_data, s); // Read a line

Reading and Writing Text Files

Characters

  • Use the get method to read a single character: char ch; input_data.get(ch);
  • Use unget for a one-character lookahead: char ch; input_data.get(ch); if ('0' <= ch && ch <= '9') // It was a digit { input_data.unget(); // Oops - didn't want to read it int n; input_data >> n; // Read integer starting with ch }
  • Older version of the library use put_back (remember last character): input_data.put_back(ch)

Reading and Writing Text Files

Stream fail

  • Stream may be in a fail state if:
    • It couldn't be opened (doesn't exist, no permission)
    • You don't have write permission on an output file
    • The stream has discovered the end of an input file
  • The fail function tells you when input has failed
  • Close the file when you are done with it: input_data.close()

Reading and Writing Text Files

Output

  • Writing to a file is similar
  • Use an ofstream object: ofstream output_data; output_data.open("output.dat");
  • Send information: output_data << n << " " << x << "\n";
  • Write a single character: output_data.put(ch);
  • Remember to close the file: output_data.close();

Reading and Writing Text Files

  • open expects a char * (a C-style string) ifstream input_data; input_data.open("input.dat");
  • If filename is in a string variable, use c_str(): string input_name; cin >> input_name; ifstream input_data; input_data.open(input_name.c_str());
  • Filenames may contain path information: ~/homework/input.dat // UNIX
  • Backslashes (used in Windows) in a string constant need to be escaped: input_data.open("c:\\homework\\input.dat");

Stream Manipulators

  • Found in the iomanip header file
  • Allow you to format output
  • Objects that are sent to a stream use the << manipulator
  • setprecision() is an example: out << setprecision(2);
    • Does not change the stream (add any characters)
    • Affects how subsequent float-type outputs are written

Stream Manipulators

  • Default for float-type is general format
    • Displays # of digits set with setprecision (default is 6)
    • Switches to scientific, as needed
    out << 12.3456789 << " " << 123456789.0; yields 12.3457 1.23457e+08
  • fixed uses same # of digits after the decimal 12.345679 123456789.000000
  • All manipulators set the state of the stream object for all subsequent operations
    • Except for setw(), which resets to 0 after each output operation

String Streams

  • Use the sstream header file
  • Used to "read from" and "write to" strings (like other files)
  • Handy for reading a number in a string, or converting numbers into strings
  • istringstream class reads data from a string
  • ostringstream class writes data to a string

String Streams — istringstream

  • E.g.: Given a date string, extract the month, day, and year string input = "January 23, 1955"; istringstream instr(input); creates an istringstream object, instr, around the input string
  • Use >> to read the month, day, comma and year: string month; int day; string comma; int year; instr >> month >> day >> comma >> year;
  • Had we used substr() to take this apart, we'd have only strings, which we would need to convert to integers

String Streams — istringstream

  • Converting strings to integers is common
  • A helper function in our private library might be handy: int string_to_int(string s) { istringstream instr(s); int n; instr >> n; return n; }
  • Can be used like this: int i = string_to_int("1999");

String Streams — ostringstream

To convert a number to a string:

  • Construct an ostringstream object: ostringstream outstr;
  • Change << to send a number to the stream: outstr << setprecision(5) << sqrt(2);
  • Use the str() function to recover the string: string output = outstr.str();

String Streams — ostringstream

  • Example: (builds the string "January 23, 1955") string month = "January"; int day = 23; int year = 1955; ostringstream outstr; outstr << month << " " << day << "," << year; string output = outstr.str();
  • Another useful helper function: string int_to_string(int n) { ostringstream outstr; outstr << n; return outstr.str(); }

String Streams

  • We commonly accept input a line at a time, then parse each line
  • String streams make this task easier
  • Avoids problems of mixing << and getline()
  • E.g., getting a time from the user. Input lines might be: 21:30 9:30 pm 9 am
  • Inputs are converted to 2 integers: HH24, and MM
  • A new "standard" string is made: HH24:MM
  • read_time() accepts the entire line, computes the time, calls time_to_string()

Command Line Arguments

From the command line, arguments may be passed to a program:

prog -v input.dat
  • prog receives two (arguments):
    1. "-v"
    2. "input.dat"
  • By convention, arguments that start w/a - are options

Command Line Arguments

  • To receive command-line arguments, main needs to look a little different: int main(int argc, char* argv[]) { ... }
  • main now accepts 2 parameters:
    • An integer — the number (count) of command-line arguments (strings) on the command line (including the program name itself)
    • An array of C-style strings (character arrays) — the value of the arguments
  • Example: argc is 3, and argv contains three strings: string(argv[0]): "prog" string(argv[1]): "-v" string(argv[2]): "input.dat"

Command Line Arguments — caeser.cpp

  • caeser is a program which will encrypt text files
  • A simple rotation cipher is used

    Given key k (1 ≤ k ≤ 25), shift each letter k spaces.

    If k is 3 (the default):

    • A → D
    • a → d
    • b → e
    • x → a
    • z → c
  • This type of cipher was used by Julius Caeser

Random Access

  • You can access any position by moving the file pointer prior to a read or write operation
  • Previous programs were examples of sequential access – reading (or writing) a byte (or other unit) at a time
  • Random access allows us to access specific locations, and modify only those locations
  • Only file streams (disk files) support random access
    • cin and cout, do not
  • (There is nothing random actually happening, nor happening randomly)

Random Access

Each disk file has two special positions:

get pointer
Location of the next character to be read
Normally starts at the beginning of the file
put pointer
Location of where the next character will be written
Normally at the end of the file (output is appended)
If moved prior to the end, overwrites existing contents

Figure 4 Sequential and Random Access

Random Access

  • To move the get and put pointers to character n positions from the beginning of the file fs: fs.seekg(n, ios::beg); fs.seekp(n, ios::beg);
  • The second argument can also be ios:end or ios::cur
  • To determine the current position of the get and put pointers (from the beginning of the file), use n = fs.tellg(); n = fs.tellp();
  • Because the files can be very large, the positions are long integers
  • To find the number of characters in a file, move the get pointer to the end, then find its offset (from the beginning): fs.seekg(0, ios::end); long file_length = fs.tellg();

Random Access

  • When manipulating data, pay attention to the formatting
  • Example: Suppose we store the data as text:
     
    H a c k e r , H a r r y 3 4 5 0 0 \n C r a c k e r

    Now raise Harry's salary by 5.5 percent to $36,397.50:

    H a c k e r , H a r r y 3 6 3 9 7 . 5 r a c k e r

    The update overwrites some characters in the next field

  • Random access is better suited to files with fixed-length records (and fixed-length fields)

Random Access

Fixed-Length Records

  • Every record in the file has the same size
  • Easy to jump over records
  • A given field has the same size in all records
  • Fields are in the same order in all records
  • Easy to find a field in a record
  • Data can be safely modified in-place

Advanced Topic

Binary Files

  • Binary files are raw data (jpeg, mp3, etc.)
  • Text files are meant to be human readable
    • The newline is messy
    • Could be \n (Unix), \r\n (Windows, Vax), or just \r (Mac)
    • All files in unix are "binary"
  • By default, the C++ I/O library opens files in text mode
  • Numbers, written as characters take up more space
    • 1 byte is needed for each digit (base 10)
    • 1 byte could contain a base-256 digit
    • Tough to debug, for a human to look at the files

Random Fact

Databases

  • Databases handle truly massive amounts of data
  • They do it fast
  • Most follow the relational model
    • Data organized into different files (tables)
    • Store your home address once
    • A different table lists all of your accounts (once)
    • A 3rd table might record all transactions on a given account
    • Information across tables are linked by a key

Random Fact (cont.)

Privacy

  • Many organizations like to use your Social Security Number (SSN)
  • It's become a universal ID (counter to its original purpose)
  • It's a poor choice for and ID (and for a key)
    • There is no checksum
    • Easy to make up a number
    • Possible that the same number is assigned to more than one person (illegal immigrants)
  • Many people are concerned that everyone wants your SSN
  • Don't give it out! Those that are required:
    • IRS, and other government agencies
    • Any bank that holds your money (to report income)
    • Your employer
  • Finally, consider merging various databases
    • Insurance companies might charge more if you have relatives with a certain disease
    • You might be denied a loan (or a job) because of an inaccurate credit or medical report. They won't even tell you why.
