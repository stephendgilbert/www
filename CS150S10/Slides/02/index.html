Number Types

  • Integer numbers (int) are whole numbers without a fractional part
    • Includes zero and negative numbers
    • Used for storing values that are conceptually whole numbers (e.g. pennies)
    • Process faster and require less storage space
  • Floating-point numbers (double) have decimal points

  • Integer arithmetic is exact
  • Floats are approximations
  • int implies value is naturally a whole number
  • int takes less space than double
  • Integer processing is faster
  • Floats can store numbers from a much larger range

  • ints have a limited range

Input

>> (stream-insertion operator)

  • reads console input: cin >> pennies;
  • Formats input. Here, converts input to an int

Common Error

Failed input

>> operator not well-suited for unexpected user input. Consider coins2.cpp, where the user inputs 10.75:

cin >> pennies;
  • Consumes the 10 ( . isn't part of an int )
  • Input stream now contains .75
cin >> nickels;
  • . not part of an int. Nothing is read
  • nickels retains previous value
  • cin is now in a "fail" state
  • All without warning

Assignment

= (Assignment Statement)

  • = operator assigns the RHS value to the variable on the left total = pennies * 0.01;
  • LHS must be an L-val, a non-const variable
  • Does not store formulas; only values
  • Assignment is not algebraic equality

Assignment

  • RHS evaluated first, to a single value, then stored in the LHS: total = count * 0.05 + total;

Assignment

Types

  • LHS and RHS types should match float total; total = "a lot"; is an error
  • Some coercions can be done by the compiler. E.g., assigning an int into a float type: total = count
  • Assigning a float type into an int causes truncation, loss of data
    • Should generate a warning

Assignment

Definition, initialization

  • double total = count * 0.01; is a definition of total
  • Creates a new variable, declares it of type double
  • Initializes it with count * 0.01
  • Can not redefine a variable (in the same scope)

Assignment

Increment and Decrement Operators

  • Assignments such as month = month + 1; are very common in programming
  • This assignment increments the variable month
  • C++ provides special shorthands for increment: month++; // add 1 to month ++month; // same thing, if used alone
  • C++ also supports decrement: month--; // subtract 1 from month --month;

Common Error

Roundoff Errors

  • Float types are approximations; not always exact (consider 1/3)
  • They are truncated when cast to an int, not rounded
  • Leads to rounding errors. E.g., int n = 4.35 * 100
    • 4.35 * 100 is a double
    • n is an int
    • n stores the value 434
  • Add 0.5 to a number before converting to an integer

Constants

  • Descriptive identifiers make a program easier to read

Arithmetic

  • Arithmetic operators in C++, by decreasing precedence
    • Multiplication, division, modulus: * / %
    • Addition, subtraction: + -
    Note, there is no exponentiation operator
  • After precedence is applied, all of these operators associate left-to-right
  • Parentheses force evaluation
  • Parentheses can make code more legible

Arithmetic

  • 4 + 6 / 2 is 7
  • (4 + 6) / 2 is 5
  • 2 * 3 + 5 is 11
  • 2 * (3 + 5) is 16
  • 8 - 3 - 2 is 3
  • 8 - (3 - 2) is 7

Arithmetic (Integer Division)

  • Integer division (and multiplication) is different from float arithmetic
  • An integer divided by an integer is an integer
  • From grade school: 11/4 is 2r3 (2 remainder 3), so, in C++: 11 / 4 // 2
  • In division, if either of the operands is a floating-point number, floating-point division is used: 11.0 / 4.0 // 2.75 11 / 4.0 // 2.75 11.0 / 4 // 2.75

Common Error

Integer Division

This is an easy thing to forget about. Consider calculating the distance an object dropped from rest falls after t seconds:

d = 1 / 2 * 9.8 * t * t
  • Invariably 0
  • Remember, evaluated left-to-right (given equal precedence)
  • You must write d = 1.0 / 2 * 9.8 * t * t or d = 1 / 2.0 * 9.8 * t * t

Common Error (cont)

Integer Division

  • You can use a caste: d = static_cast<double>(1) / 2 * 9.8 * t * t
  • But notice the error here: d = static_cast<double>(1 / 2 * 9.8 * t * t )
  • The expression inside the parentheses (1 / 2 * 9.8 * t * t) is evaluated first, to 0
  • This value is then cast to a double, yielding 0.0

Arithmetic — division, modulus

  • To get the remainder of division between two integers, use the modulus operator %: 7 % 4 // 3 remainder
  • Modulus is undefined on float-types
  • Suppose we want to know the value of coins in a purse, in dollars and cents
    • Compute total cents, as an integer
    • Divide to get the whole dollar amount dollars = value / 100
    • Compute the remaining change cents = value % 100

Quality Tip

Factor Out Common Code

Consider the Quadratic Formula:

  • We have no ±, so we might code this: x1 = (-b + sqrt(b * b - 4 * a * c)) / (2 * a); x2 = (-b - sqrt(b * b - 4 * a * c)) / (2 * a);
  • Duplicating code, sqrt(b * b - 4 * a * c)
    • Wasteful
    • Error-prone
  • Factor out common code: double root = sqrt(b * b - 4 * a * c); x1 = (-b + root) / (2 * a); x2 = (-b - root) / (2 * a);

Strings

String Variables

Strings

  • Use the getline() function: getline(cin, name); to read all keystrokes until the newline (the [Enter] key)
    • The newline is extracted from the stream, but not stored with the string
  • If the user enters Harry Hacker then name will store the string "Harry Hacker"

Strings

Member Functions

  • The number of characters in a string is its length
  • You can ask a string its length by using its length() method, or member function
  • Invoked using dot notation name.length()
  • Much like sending a message to an object

Substrings

  • The substr member function is used to extract substrings s.substr(start, length)
  • Result is length characters from s, starting at index start
  • Indices are zero-based (first character is at index 0)
  • The last character is always one less than the length of the string

Using Objects
