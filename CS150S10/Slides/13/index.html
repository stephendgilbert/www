Sets

  • Vectors and lists keep elements in order
  • Sets don't maintain an order
  • Tradeoff: lose ordering, but find things quickly
Set
Unordered collection of distinct items
  • E.g.:
    • Called numbers in a Bingo round
    • Dictionary, for a spell-checker
    • List of available printers on a network
    • List of invitees

Operations on a Set

  • Add an element
    • Adding an existing element has no effect
  • Remove an element
    • Removing a non-existent element has no effect
  • Find an element
  • Traverse all elements

C++ set (in the STL)

  • It's a template: #include <set> set<string> names;
  • Use insert() and erase(): names.insert("Romeo"); names.insert("Juliet"); names.insert("Romeo"); // Has no effect: "Romeo" is already in the set names.erase("Juliet"); names.erase("Juliet"); // Has no effect: "Juliet" is no longer in the set
  • To check for existence: int c = names.count("Romeo"); // count returns 1

STL set (iterators)

  • You can visit each element: names.insert("Tom"); names.insert("Dick"); names.insert("Harry"); set<string>::iterator pos; for (pos = names.begin(); pos != names.end(); pos++) cout << *pos << " ";
  • They are returned in sorted order (not insertion order): Dick Harry Romeo Tom

The STL multiset

  • A multiset (also called a bag) may contain duplicates
  • Element count is maintained multiset<string> names; names.insert("Romeo"); names.insert("Juliet"); names.insert("Romeo"); // Now names.count("Romeo") is 2 names.erase("Juliet"); // Now names.count("Juliet") is 0 names.erase("Juliet"); // Has no effect: "Juliet" is no longer in the bag

Binary Search Trees

Sets can be represented in a number of ways

  • Ordered array:
    • O(log(n)) lookups (binary search)
    • O(n) insertion/removal
  • Unordered array:
    • O(1) insertion/removal
    • O(n) lookups
  • Binary search tree:
    • Simplest of the many, more efficient, set data structures
    • O(log(n)) operations (if tree is properly maintained)

Binary Trees

Binary trees have the following properties:

  • Finite collection of nodes
  • Each node has, at most, two children
  • Each node has exactly one parent
    • Except for the root node
  • If balanced, the height is O(log(n))
    • There is a path of length < log(n) from the root to every node in the tree
  • Note, a subtree of a binary tree is a binary tree

Binary Search Trees

A binary search tree is a binary tree, with the following properties:

  • The values of all descendants to the left of any node are less than the value of that node, and
  • All descendants to the right have greater values

Binary Search Tree (Inserting)

Simple algorithm for inserting into a BST:

  • If node is non-NULL, look at its data
    • If its value is larger than the one you want to insert, go left
    • If its value is smaller, go right
  • If you encounter a NULL node, replace it with the new node
BinarySearchTree tree; tree.add("Juliet"); tree.add("Tom"); tree.add("Dick"); tree.add("Harry");

Binary Search Tree – TreeNode::insert_node()

Helper function for insert() (does most of the actual work)

void TreeNode::insert_node(TreeNode* new_node) { if (new_node->data < data) { if (left == NULL) left = new_node; else left->insert_node(new_node); } else if (data < new_node->data) { if (right == NULL) right = new_node; else right->insert_node(new_node); } }
  • Checks if new object is less than node's value
    • If so, recursively calls its left child's insert_node
    • Else, recursively calls its left child's insert_node
  • If the desired child is null, makes a child of the new node

Binary Search Tree – BinarySearchTree::insert()

  • "Wraps" data in a new node
  • If tree is empty (root is NULL), new node becomes root
  • Otherwise, call root's insert_node method void BinarySearchTree::insert(string data) { TreeNode* new_node = new TreeNode; new_node->data = data; new_node->left = NULL; new_node->right = NULL; if (root == NULL) root = new_node; else root->insert_node(new_node); }

Binary Search Tree (Removal)

To remove a element from the set:

  • First find the node
  • If it has no child trees, remove it
    • Set the parent link to this node to NULL
  • If it has only 1 child, promote the entire subtree
    • Modify the link from the parent so that it points to the child

Binary Search Tree (Removal)

If the node to be removed has 2 children:

  • It is simplest to replace the node with its smallest right descendant
  • This successor is the very next larger value
  • Maintains BST property
  • Easy to remove, since it has, at most, one child
    • Successor is left-most node in the right subtree
  • Successor is a single node, and replaces target

Binary Search Tree (Runtime)

  • Lookups (and insertion and removal) are O(h), the height of the tree

Balanced Trees

  • balanced – each node has approximately as descendants on the left as on the right
  • Height of tree is O(log(n))

Unbalanced (Degenerate) Trees

  • Height grows more quickly as tree becomes unbalanced
  • Consider, a linked list is a binary tree
  • Worst case, height is O(n)

Tree Traversal

  • Use an inorder traversal to print elements if a BST in order:
    1. Print the left subtree
    2. Print the node's data
    3. Print the right subtree
  • Recursive algorithm
  • Visits each node in a BT exactly once

Tree Traversal – print()

void TreeNode::print_nodes() const { if (left != NULL) left->print_nodes(); cout << data << "\n"; if (right != NULL) right->print_nodes(); } void BinarySearchTree::print() const { if (root != NULL) root->print_nodes(); }

Tree Traversal

There are other schemes, useful for other applications of binary trees:

preorder traversal ("me first")

  1. Print the node's data
  2. Print the left subtree
  3. Print the right subtree

postorder traversal ("me last")

  1. Print the left subtree
  2. Print the right subtree
  3. Print the node's data

Maps

  • Stores associations between keys and values
  • Keys are unique
  • Also called a dictionary
  • Is a set of key-value pairs

Figure 10 A Map

Maps – map class

  • Declared in the <map> header file
  • Takes 2 type arguments: the key type, and the value type
    • Requires that the < operator be define on the key type
  • Use the [] operator to create associations: map<string, double> scores; scores["Tom"] = 90; scores["Dick"] = 86; scores["Harry"] = 100;
  • Also used to read values: cout << "Tom's score: " << scores["Tom"];

Maps – map class

  • find() returns an iterator
  • The iterator "points to" a pair
    • Declared in <utility>
    • Has 2 public fields: first and second
    map<string, int>::iterator pos = scores.find("Harry"); // Call find if (pos == scores.end()) // Check if there was a match cout << "No match for Harry"; else cout << "Harry's score: " << (*pos).second;

Maps – map class

  • As with pointers, (*pos). can be written pos->
  • Code to iterate over the contents of a map: map<string, int>::iterator pos; for (pos = scores.begin(); pos != scores.end(); pos++) { cout << "The score of " << pos->first << " is " << pos->second << "\n"; }

Maps – multimap

  • Keys need not be unique
  • Instead of [], use insert() and erase()
    • make_pair() function also in <utility>
  • E.g. multimap<string, string> friends; friends.insert(make_pair("Tom", "Dick")); // Dick is a friend of Tom friends.insert(make_pair("Tom", "Harry")); // Harry is also a friend of Tom

Maps – multimap

  • A "find" operation essentially describes a range, which can be iterated over
  • First, get the range (for a given key): multimap<string, string>::iterator lower = friends.lower_bound("Tom"); multimap<string, string>::iterator upper = friends.upper_bound("Tom");
  • Then iterate over that range: cout << "Tom's friends: "; for (multimap<string, string>::iterator pos = lower; pos != upper; pos++) cout << pos->second << " ";
  • To remove a single key-value pair, pass an iterator to erase() friends.erase(pos);

Priority Queues

A collection organized so as to permit fast access to and removal of the largest element

  • Prioritization is a weaker condition than ordering
  • Order of insertion irrelevant
  • Element with the highest priority (whatever that means) comes out next
  • Not really a queue: not a FIFO
  • Basic operations:
    • push() – Adds an element to the collection
    • top() – Looks at the element with the highest priority
    • pop() – Removes this element

Advanced Topic

Discrete Event Simulations

  • Classic application of a priority queue
  • Events are prioritized by a timestamp (lower timestamp has higher priority) class Event { public: double get_time() const; virtual void act(); ... };
  • The "clock" (priority queue) is loaded with some initial events
  • Simulator processes events in order
  • Events may spawn other events, which are added to the priority queue

Advanced Topic

Discrete Event Simulation (cont.)

  • Each type of event (arrival, departure, etc.) is a subclass of Event
    • Each subclass overrides the act() method
  • The heart of the simulator is the event loop
    • Removes next even from the priority queue
    • Set the simulation clock to that event's time
    • Calls that event's act() method
    while (event_queue.size() > 0) { Event* next_event = event_queue.top(); current_time = next_event->get_time(); next_event->act(); // Typically adds new events delete next_event; }

Advanced Topic

Discrete Event Simulation (cont.)

  • The event queue holds Event*
  • The < operator is already defined
  • We will define a function to use to compare events: bool event_less(const Event* e1, const Event* e2) { return e1->get_time() > e2->get_time(); // The earliest event should have the largest priority }
  • We tell the priority_queue to use this function, instead (see Chapter 20): priority_queue<Event*, vector<Event*>, bool (*)(const Event*, const Event*)> event_queue(event_less);

Heaps (Cost)

  • Heap keeps a balanced shape
  • Insertion and removal each require, at most, h swaps
  • Tree of height h contains at least 2h-1 nodes, but less than 2h nodes
  • Then h-1 ≤ log2(n) < h
  • So, insertion and removal require O(log(n)) steps
