Analyzing the Performance of the Selection Sort Algorithm

  • Number of operations is tough, and differs across computers
  • Count the number of times an element is visited
  • Consider an vector of length n
  • Find the smallest of n elements
    • Requires n visits
  • Swap smallest into place
    • Requires 2 more visits
  • Repeat for n-1 elements
  • Again, for n-2, etc., until array is sorted

Analyzing the Performance of the Selection Sort Algorithm

  • We're interested in ratios; how the function grows
  • As n gets large, the lower-order terms don't contribute much
  • The leading coefficient divides out. Ignore it
  • We say, "The number of visits is of order n2"
  • big-oh notation: The number of visits O(n2)
  • Doubling the size of the vector quadruples the time required to sort (roughly)
  • An increase of inputs by a factor of 100 raises the runtime by a factor of 10,000
  • Luckily, there are faster sorting algorithms

Analyzing the Performance of the Selection Sort Algorithm

Big-O

  • If f(n) is O(g(n)), then f grows no faster than g
    • It might grow the same (roughly)
    • It might grow much more slowly
    • g dominates f
  • f(n)/g(n) is less than some constant C, as n gets arbitrarily large

Merge Sort

  • Divide and conquer algorithm
    • Split array into 2 halves
    • Get somebody to sort each half
    • Halves can be merged into a single, sorted array, in linear time
  • Recursively use the mergesort to sort each half
    • As a base case, a subarray with 0 or 1 elements is sorted

Analyzing the Merge Sort Algorithm

  • The merge sort seems to be a tremendous improvement
  • To understand why, estimate the number of element visits
  • Consider the merge portion of the algorithm
    • Each step adds another element to b
    • Compare the first two elements of the halves and decide which is smaller (2 visits)
    • Copy the smaller element to its place in the sorted vector (1 visit)
    • Copy the sorted vector back into the original vector (2 visits)
  • 5 visits for each of the n elements
  • 5n visits for a single merge

Analyzing the Merge Sort Algorithm

  • If T(n) is the time required to sort n elements, then T(n) = 2T(n/2) + 5n
  • evaluate T(n/2) T(n/2) = 2T(n/4) + 5n/2
  • Substitute in: T(n) = 2×2T(n/4) + 5n + 5n
  • Again: T(n/4) = 2T(n/8) + 5(n/4) T(n) = 2×2×2T(n/8) + 5n + 5n + 5n

Analyzing the Merge Sort Algorithm

  • Generalizing: T(n) = 2kT(n/sk) + 5nk
  • Assume n = 2m, let k=m : T(n) = 2mT(n/sm) + 5nm = nT(1) + 5nm = n + 5n×log2(n)
  • Hence merge sort is an O(nlog(n)) algorithm
    • Drop the lower-order terms
    • Drop the leading coefficient
    • Drop the base of the logarithm

Searching

Linear search

  • Linear/sequential search: searching through the elements of an arbitrary sequence until the match is found or until the end of the sequence for (int i = 0; i < v.size(); i++) { if (v[i] == value) return i; }
  • Visit each element at most 1 time
  • Visit all n to verify failure
  • Visit n/2 elements on average (if element exists once)
  • It is an O(n) algorithm

Searching

  • The binary search cuts the search space in half at each step
  • A recursive binary search (see bsearch.cpp) int binary_search(vector<int> v, int from, int to, int value) { if (from > to) return -1; int mid = (from + to) / 2; if (v[mid] == value) return mid; else if (v[mid] < value) return binary_search(v, mid + 1, to, value); else return binary_search(v, form, mid - 1, value); }

Searching

  • O(log(n)) run-time
  • Look at middle, then search 1/2 the array: T(n) = T(n/2) + 1 T(n/2) = T(n/4) + 1
  • So T(n) = T(n/4) + 2
  • Generally, T(n) = T(n/2k) + k
  • Let n = 2k
  • So T(n) = 1 + log2(n)

Searching

  • Binary search is faster than linear search
  • Only works on sorted arrays/vectors
  • In instances where the search is made only once, it is not worthwhile to sort the data first just to use the binary search

Library Functions for Sorting and Binary Search

  • C++ provides functions in the C++ library for sorting and searching
  • In the <algorithm> header
  • To sort vector v call sort(v.begin(), v.end());
  • To sort array a sort(a, a + size)

Library Functions for Sorting and Binary Search

binary_search()

  • Returns true or false, not the index
  • Searching a sorted vector: binary_search(v.begin(), v.end(), value)
  • Searching a sorted array: binary_search(a, a + size, value)
