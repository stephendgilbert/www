Inheritance
Define new classes that are extensions of existing classes
Polymorphism
Allows you to take advantage of the commonality between related classes
Individual classes may have specific behavior
Used in building very flexible and extensible systems

Derived Classes

  • Inheritance is a mechanism for enhancing existing working classes
  • A new class inherits from a class representing a more general concept
  • A manager is a special type of employee: class Manager : public Employee { public: new member functions private: new data members };
  • base class — the more general class (Employee)
  • derived class — the more specialized class, inherits from the base class (Manager)

Derived Classes

  • All member functions and data members of the base class are inherited by the derived class Manager m; m.set_salary(68000);
  • In the derived class definition you specify only new member functions and data members
  • The class diagram shows the relationship between these classes
  • The arrow with a hollow head indicates inheritance

Syntax : Derived Class Definition

class DerivedClassName : public BaseClassName { features };

Example:

class Manager : public Employee { public: Manager(string name, double salary, string dept); string get_department() const; private: string department; };

Purpose:

Define a class that inherits features from a base class.

Derived Classes

Travel Clock

  • Shows time in a different location
  • Stores city names and relative time differences
  • The time for a TravelClock is computed by taking the local time and adding the time difference. TravelClock clock(true, "New York", 3); cout << "The time in "; << clock.get_location() << "; is "; << clock.get_hours() << ";:"; << clock.get_minutes();
  • Derive TravelClock from Clock
  • Different in three ways:
    • Stores location and time difference
    • get_hours() adjusts current time
    • get_location returns something other than "Local"

Derived Classes

  • When the TravelClock class inherits from the Clock class, it needs only to spell out these three differences. class TravelClock : public Clock { public: TravelClock(bool mil, string loc, double off); int get_hours() const; string get_location() const; private: string location; int time_difference; };

Common Error

Private Inheritance

Don't forget the public modifier when inheriting:

class Manager : Employee // Error { ... };

Defaults to private

  • Legal
  • Rarely useful
  • Employee methods in a Manager object all become private
  • These methods can't be called on a Manager object
  • So, a derived-class object is not usable in the same context that a base-class object is

Calling the Base-Class Constructor

The constructor of a derived class has two tasks:

  • Initialize the base object
  • Initialize all data members

Initializing data members

  • Fairly straightforward
  • We'll avoid negative values by adding 24 TravelClock::TravelClock(bool mil, string loc, int diff) // Not complete { location = loc; time_difference = diff; while (time_difference < 0) time_difference = time_difference + 24; }

Calling the Base-Class Constructor

Initializing the base class is less straightforward

  • We must set Clock::military
  • Clock has no methods for doing this
  • Must be done through the constructor
  • Frequently, we must invoke the base-class constructor before initializing the derived-class data
  • Use the initializer list: TravelClock::TravelClock(bool mil, string loc, int diff) : Clock(mil) { ... };
  • If you omit the base-class constructor, then the base object is constructed using its default constructor (if it has one)

Syntax : Constructor with Base-Class Initializer

DerivedClassName :: DerivedClassName(expressions) : BaseClassName(expressions) { statements }

Example:

Manager::Manager(string name, double salary, string dept) : Employee(name, salary) { department = dept; }

Purpose:

Supply the implementation of a constructor, initializing the base class before the body of the derived-class constructor.

Overriding Member Functions

  • It is common to override a member function of a base class
  • TravelClock::get_hours() needs to adjust the local time for the location int TravelClock::get_hours() const { int h = local hour value; if (clock uses military time) return (h + time_difference) % 24; else { h = (h + time_difference) % 12; if (h == 0) return 12; else return h; } }
  • We need to determine whether the clock uses military time, and what the time is

Overriding Member Functions

  • Can't look at Clock::military ; it's private
  • Use Clock::is_military() if (is_military()) ...
  • It's inherited from Clock, so call it through the implicit parameter of the TravelClock
  • To get hours, we need the base class' get_hours()
  • However, we can't use the implicit parameter; both the base class and the derived class have a get_hours() method
  • Need to explicitly scope it to the base class: int TravelClock::get_hours() const { int h = Clock::get_hours(); ... }

Overriding Member Functions

In general, if B::f is a function in the base class, then the derived class D can take three kinds of actions:

  • The derived class can extend B::f by supplying a new implementation that refers to the base-class implementation.
    For example, the TravelClock::get_hours function is an extension of Clock::get_hours
  • The derived class can replace B::f by supplying a new implementation that is unrelated to the base-class implementation.
    For example, TravelClock::get_location is a replacement for Clock::get_location
  • The derived class can inherit B::f by not supplying an implementation for f.
    For example, TravelClock inherits Clock::get_minutes and Clock::is_military

Advanced Topic

Protected Access

  • C++ provides a third access modifier: protected class Clock { public: ... protected: bool military };
  • Protected members can be accessed by all derived classes
  • Still hidden from the outside world
  • This would alleviate some of the grief experienced when implementing TravelClock

Advanced Topic

Protected Access (continued)

Protected members are still subject to the same problems that public members present

  • Base class designer has no control over authors of the derived classes
  • Derived-class methods can corrupt base-class data
  • Data-hiding is broken; the base class becomes harder to modify

It is better to make all attributes private

  • To provide access only to derived classes, provide protected accessor methods

Polymorphism

  • You will find that there is quite a bit of repetitive code in main(), in clocks2.cpp
  • The compiler realizes that a TravelClock is a special case of a Clock, and permits the assignment clocks[1] = TravelClock(true, "Rome", 9);
  • However, there is no room to store the derived-class data
  • That data simply gets sliced away when a derived-class object is assigned to a base-class variable.

Figure 4 Slicing Away Derived-Class Data

Polymorphism

  • TravelClock objects are larger than Clocks
  • Vectors can't handle this
  • Store pointers (or references) in the vector, instead
  • A base-class pointer can point to a derived-class object. (The reverse is an error)
  • Pointers to the various clock objects all have the same size - the size of a memory address - even though the objects themselves may have different sizes
  • Since every TravelClock is a special case of a Clock, the starting address of a TravelClock object is the starting address of a Clock object

Polymorphism

Code to store both base-class and derived-class objects:

vector<Clock*> clocks; // Populate clocks clocks[0] = new Clock(true); clocks[1] = new TravelClock(true, "Rome", 9); clocks[2] = new TravelClock(false, "Tokyo", -7);

Figure 5 A Polymorphic Vector

Polymorphism

The vector now holds pointers, so, the code to print needs to be changed a bit

cout << clocks[i]->get_location() << " time is " << clocks[i]->get_hours() << ":" << setw(2) << setfill('0') << clocks[i]->get_minutes() << setfill(' ') << "\n";

Unfortunately, the following code does still not work! The output is still

Local time is 21:15 Local time is 21:15 Local time is 9:15
  • Recall that both Clock and TravelClock have a get_location and a get_hours member function
  • The compiler calls the Clock member function because the pointer is of type Clock*

Polymorphism

  • We want to find the actual type to which clocks[i] points
  • Then call the appropriate methods
  • You must alert the compiler that the function call needs to be preceded by the appropriate function selection
  • Use the virtual keyword: class Clock { public: Clock(bool use_military); virtual string get_location() const; virtual int get_hours() const; int get_minutes() const; bool is_military() const; private: . . . };

Polymorphism

  • Now, each object's type will be inspected, at run time
  • Such a selection/call combination is called dynamic binding
  • The traditional call, which always invokes the same function, is called static binding (is known at compile time)
  • virtual must be used in the base class
  • Applies to all functions of the same name and parameter types in the derived class
  • Considered good form to explicitly label these as virtual, too

Syntax : Virtual Function Definition

class ClassName { virtual return_type function_name(parameter1, parameter2, ..., parametern); ... };

Example:

class Employee { public: virtual double get_salary(); ... };

Purpose:

Define a dynamically bound function that can be redefined in derived classes. When the function is called, the actual type of the implicit parameter determines which version of the function executes.

Common Error

Slicing an Object

  • It is legal to copy a derived-class object into a base-class variable
  • Derived-class data is lost Manager m; ... Employee e = m; // Holds only the Employee base data of m
  • This may be desirable
  • If not, use pointers
  • The reverse assignment is not legal: Employee e; ... Manager m = e; // Error

Advanced Topic

Virtual Self-Calls

  • Add a print function to Clock: void Clock::print() const { cout << get_location() << " time is " << get_hours() << ":" << setw(2) << setfill('0') << get_minutes() << setfill(' ') << "\n"; }
  • This call works just like you'd want it to: TravelClock rome_clock(true, "Rome", 9); rome_clock.print();
  • The call to get_hours uses the implicit parameter: this->get_hours()
  • It's virtual, so, the right class' method is called
