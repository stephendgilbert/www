Intro

Recursion is:

  • When the same computation recurs, or occurs repeatedly, as the problem is solved
  • A technique that breaks up complex computational problems into simpler ones

Triangle Numbers (Larger Triangle)

  • To deal with general case, consider this picture: [] [][] [][][] [][][][]
  • Knowing the area of the smaller, colored triangle, the area of the larger triangle is: smaller_area + width
  • To find the smaller area, make a smaller triangle and ask it for its size: Triangle smaller_triangle(width - 1); int smaller_area = smaller_triangle.get_area();

Common Error

Infinite Recursion

  • A function calling itself over and over with no end in sight
  • Exhausts all bookkeeping memory
  • Program shuts down and reports a "stack fault"
  • Happens because the parameter values didn't get simpler or a terminating case is missing

Permutations

  • A rearrangement of the letters
  • Function that generates all permutations of a word: vector<string> generate_permutations(string word)
    • Iterates through the character positions of the word
    • Each loop iteration creates a shorter word that omits the current position
    • Compute the permutations of the shorter word
    • For each of the shorter permutations, add the omitted letter
  • Algorithm is recursive — uses the fact that we can generate the permutations of shorter words
    • Stopping point is word of length one. Permutation is only itself

Thinking Recursively

Step 1

  • Goal is to implement the predicate function bool is_palindrome(string s)
  • Consider ways to simplify the inputs so that the same problem can be applied to the simpler input
  • In our example, the input is the string that we need to test
  • Possible ways to simplify:
    • Remove the first character
    • Remove a character from the middle
    • Cut the string into two halves
  • To find a recursive solution, reduce the input to a simpler input for the same problem

Thinking Recursively

Step 2

  • Combine solutions with simpler inputs to a solution of the original problem
  • Consider solutions from Step 1. How can you apply the solution for the simpler inputs to current problem?
  • In the palindrome problem, you cannot cut the string in half because you will not be able to further test for the palindromes: "rotor": "rot" and "or"
  • More promising is removing the first and last characters: "rotor": "oto"
  • Then, a word is a palindrome if:
    • The first and last letters match, and
    • The word obtained by removing the first and last letters is a palindrome

Thinking Recursively

Step 3

  • Find solutions to the simplest inputs
  • To ensure recursion comes to a stop, deal with simplest inputs separately
  • The simplest strings for the palindrome test:
    • Strings with 2 characters
    • Strings with a single character
    • The empty string
  • A string with 2 characters actually breaks in half into a single character, which is a palindromes in itself
  • Strings of length 0 or 1 are palindromes because they are the same when read backwards

Thinking Recursively

Step 4

  • Combine the simple cases and the reduction step
  • Make separate cases for the simple inputs from Step 3
  • If the input isn't one of the simplest cases, implement logic from Step 2

Recursive Helper Functions

  • slight change to original problem can help to find a recursive solution
  • Adjust the start and end parameters to skip over matching letter pairs, instead of constructing new string objects to represent the shorter strings /* Tests whether a substring of a string is a palindrome. @param s the string to test @param start the index of the first character of the substring @param end the index of the last character of the substring @return true if the substring is a palindrome */ bool substring_is_palindrome(string s, int start, int end) { // Separate case for substrings of length 0 and 1 if (start >= end) return true; if (s[start] == s[end]) // Test substring that doesn¿t contain the first and last letters return substring_is_palindrome(s, start + 1, end - 1); else return false; }

Mutual Recursion

  • In a mutual recursion, a set of cooperating functions calls each other repeatedly
  • We will develop a program that can compute the values of arithmetic expressions such as: 3 + 4 * 5 (3 + 4) * 5 1 - (2 - (3 - (4 - 5)))

Mutual Recursion

Figure 3 Syntax Diagrams for Evaluating an Expression:
  • expression — either a term, or a sum or difference of terms
  • term — either a factor, or a product or quotient of factors
  • factor — a number or expression enclosed in parentheses

Mutual Recursion

  • To compute the value of an expression, implement
    • expression_value — first calls term_value to get the value of the first term. If the next input character is + or -, it calls term_value again and does the appropriate operation
    • term_value — calls factor_value in the same way, multiplying or dividing
    • factor_value — checks if the next character is a '(' or a digit. If there is a parenthesis, it makes a recursive call to expression_value
  • The three are mutually recursive
  • At each recursive call, at least some of the characters of the input are consumed, so eventually the recursion must come to an end
