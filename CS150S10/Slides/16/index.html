Templates

  • Provides another level of abstraction
  • A template class or function can work with many different types
  • Handled at compile time (efficient)

Template Functions

  • E.g., a function to print an array of ints: void print(ostream& out, int data[], int count) { out << "["; for (int i = 0; i < count; i++) { if (i > 0) out << ","; out << data[i]; } out << "]"; }

Template Functions

Your next project requires the same for an array of doubles:

  • Cut and paste?
  • Search and replace?

Both error-prone. Every int? Use a function template

  • Function parameterized by type
  • Abstract away the differences (argument types)

Template Functions

  • Use the template prefix, template followed by a list of type parameters: template<typename T> void print(ostream& out, T data[], int count) { out << "["; for (int i = 0; i < count; i++) { if (i > 0) out << ","; out << data[i]; } out << "]"; }

Template Functions

  • T is arbitrary; any legal C identifier
  • typename (or class) indicates that the value of T will be a type
  • T could represent any type
  • Its type is inferred from the arguments of the call: int a[] = {2, 4, 5}; print(cout, a, 3); // Will use int print double b[] = {3.14159, 2.7}; print(cout, b, 2); // Will use double print string c[] = {"Fred", "Sally", "Alice"}; print(cout, c, 3); // Will use string print
  • Within the function body T can be used wherever that type would be appropriate

Syntax : Template Functions

template<typename type_var1, ..., typename type_varn> return_type function_name(parameters) { statements }

Example:

template<typename T> void print(ostream& out, T data[], int count) { out << "["; for (int i = 0; i < count; i++) { out << data[i]; if (i + 1 < count) out << ","; } out << "]"; }

Purpose:

Define a function that can be reused for different types of arguments.

Common Error

Invalid Type Parameters

A type argument is characterized implicitly, by its use in the function.

  • The typical max function: template<typename T> T max(const T& left, const T& right) { if (left < right) return right; return left; }
  • Can't mix types; compiler can't guess programmer's intent double e = max(2, 3.13); // Error — cannot infer type parameter
  • Parameters may be explicitly provided: double e = max<double>(2, 3.13); // OK — integer will be converted into double

Common Error

  • The < operator must be defined for T
  • This won't work: Employee mary("Mary Smith", 25000); Employee fred("Fred Jones", 37500); Employee big = max(mary, fred); // Error - invalid types
  • Compiler must see the entire definition of a function template

Compile-Time Polymorphism

  • Templates are instantiated in many different ways
  • Does its work at compile time
    • Compiler creates template functions from the function template, as needed
    • Few run-time costs
    • A function template prototype is meaningless
  • inheritance and overriding occur at runtime
  • Templates are commonly stored in included files
  • Not compiled separately
  • Compiler looks for non-templates first

Advanced Topic

Templates and Overloading

Compiler will always choose a non-template function over a template function

  • Specialization
  • Allows special behavior to be supplied
  • << not defined on Product
  • But, Product has a print() void print(ostream& out, Product data[], int count) { out << "["; for (int i = 0; i < count; i++) { if (i > 0) out << ","; data[i].print(out); } out << "]"; }
  • When the second argument is a Product array, this will be chosen over the template that could be instantiated

Template Classes

  • Container classes are most common examples
  • Consider a function that finds the max and min values of a vector Pair minmax(vector<int> v) { int min = v[0]; int max = v[0]; for (int i = 1; i < v.size(); i++) { if (v[i] < min) min = v[i]; if (v[i] > max) max = v[i]; } return Pair(min, max); }
  • Make a Pair class to return both values

Template Classes

class Pair { public: Pair(int a, int b); int get_first() const; int get_second() const; private: int first; int second; }; inline Pair::Pair(int a, int b) { first = a; second = b; } inline int Pair::get_first() const { return first; } inline int Pair::get_second() const { return second; }

Template Classes

  • To use: Pair p = minmax(data); cout << p.get_first() << " " << p.get_second() << "\n";
  • Pair is not very flexible
    • We'd like a generic pair
    • One that can hold disparate types
  • Type arguments can not be inferred
  • Types are explicitly named at declaration: Pair<int, int> Pair<string, int>

Template Classes

  • Replace types with F and S template<typename F, typename S> class Pair { public: Pair(const F& a, const S& b); F get_first() const; S get_second() const; private: F first; S second; };

Template Classes

  • Turn each member function into a template: template<typename F, typename S> inline Pair<F, S>::Pair(const F& a, const S& b) { first = a; second = b; } template<typename F, typename S> inline F Pair<F, S>::get_first() const { return first; } template<typename F, typename S> inline S Pair<F, S>::get_second() const { return second; }

Template Classes

  • Pair is a template, not a class
  • Pair<int, string>, e.g., is a class
  • Compare to the STL pair class
  • These are templates, not functions
  • Template class definition and function definitions must be included in each compilation module

Syntax : Class Template Definition

template<typename type_var1, ..., typename type_varn> class ClassName { features };

Example:

template<typename F, typename S> class Pair { public: Pair(const F& a, const S& b); F get_first() const; S get_second() const; private: F first; S second; };

template<typename type_variable> modifiers return_type ClassName<type_variable>::function_name(parameters) constopt { statements }

Turning a Class into a Template

  • Recall the linked list from Chapter 12
  • List stored strings
  • Used the Node and Iterator classes
  • Let's make them into templates
  • Start with Node:
    • Replace each string with a T
    • Insert preamble
    • Change each Node to a Node<T> (except constructor names)

Turning a Class into a Template — Node

template<typename T> class Node { public: Node(T s); private: T data; Node<T>* previous; Node<T>* next; };

Turning a Class into a Template — List

template<typename T> class List { public: List(); void push_back(T s); void insert(Iterator<T> pos, T s); Iterator<T> erase(Iterator<T> pos); Iterator<T> begin(); Iterator<T> end(); private: Node<T>* first; Node<T>* last; };

Turning a Class into a Template

  • Turn each member function definition into a template: template<typename T> Iterator<T> List<T>::begin() { Iterator<T> iter; iter.position = first; iter.last = last; return iter; }
  • Remember, these all go into a header file
  • There is no list.cpp file

Common Error

Templates Don't Preserve Inheritance

  • A relationship between 2 type arguments does not imply a relationship between the corresponding template classes
  • E.g. Manager inherits from Employee
  • vector<Manager> is not related to vector<Employee>

Quality Tip

Document Template Parameter Requirements

  • Requirements that type arguments must satisfy are implicitly defined by the operations on them
  • Do not force a client of your template to read through the code
  • Code for member functions is only generated if that function is called
    • E.g., Product doesn't define operator==
    • vector<Product> is okay, so long as find(), etc., aren't called
  • Place a minimal set of necessary operations in the class header comments

Quality Tip

Document Template Parameter Requirements

/** A class to hold a collection of values Type T must define a copy constructor To use the find generic algorithm you must define the equality operator == */ template<typename T> class vector { ... }; /** A collection of type value pairs. type K must have a copy constructor and the less than operator. type V must have a copy constructor and the equality operator. */ template<typename K, typename V> class map { ... };

Nontype Template Arguments

Template parameters can be type names, or constants

Consider our matrix case study

  • 3 x 3, at first
  • Managed heap memory to remove this restriction
  • Template arguments provide a third alternative

Nontype Template Arguments

  • Size could be specified by integer template arguments: template<typename T, int ROWS, int COLUMNS> class Matrix { . . . private: T data[ROWS][COLUMNS]; };
  • Note, the integer values must be known at compile time

Nontype Template Arguments

  • Supply the type and the size: Matrix<double, 3, 4> a; // A 3 x 4 matrix of double values Matrix<string, 2, 2> b;
  • The bounds become part of the type
  • Compatibility checked at compile time Matrix<int, 3, 4> a; Matrix<double, 3, 4> b; Matrix<int, 5, 7> c; Matrix<int, 3, 4> d; b = a; // Error, element types don't match. c = a; // Error, sizes dont match, so types differ. d = a; // OK. Element types and sizes match.
  • Requirements for matrix multiplication are more difficult to specify

Setting Behavior Using Template Arguments

  • Template parameters can specify behavior (set policy)
  • Generalize the max function to support types that:
    • Don't define operator<
    • Have more than one way of ordering
  • Make comparison a template argument: template<typename T, typename CMP> T max(const T& left, const T& right, CMP cmp) { if (cmp(left, right)) return right; return left; }

Setting Behavior Using Template Arguments

  • < has been replaced by a function call
  • cmp may be either
    1. An actual function
    2. A function object

Setting Behavior Using Template Arguments

  • E.g., use function objects to compare Employees by salary or name
  • Develop these two classes: class CompareBySalary { public: bool operator()(const Employee& a, const Employee& b) const; }; bool CompareBySalary::operator()(const Employee& a, const Employee& b) const { return a.get_salary() < b.get_salary(); }

Setting Behavior Using Template Arguments

class CompareByName { public: bool operator()(const Employee& a, const Employee& b) const; }; bool CompareByName::operator()(const Employee& a, const Employee& b) const { return a.get_name() < b.get_name(); }

Setting Behavior Using Template Arguments

  • CompareByName and CompareBySalary are types
  • Represent two different orderings
  • We can use the max function in either sense: Employee alice("Alice Smith", 45000); Employee fred("Fred Jones", 38500); Employee one = max(alice, fred, CompareBySalary()); Employee two = max(alice, fred, CompareByName());

Setting Behavior Using Template Arguments

  • Same idea applies to template class arguments
  • Consider maintaining a collection in sorted order:
    • Specify the rules for comparing two elements
    • Do not specify criteria in the container itself
    template<typename T, typename CMP> class OrderedCollection { public: typedef Iterator<T> iterator; void add(T value); iterator begin(); iterator end(); private: List<T> data; CMP comparator; };

Setting Behavior Using Template Arguments

  • The comparator is an attribute template<typename T, typename CMP> void OrderedCollection<T, CMP>::insert(T value) { iterator ptr = begin(); while (ptr != end()) { if (comparator(value, *ptr)) { // Found place to insert data.insert(ptr, value); return; } ++ptr; } // Insert at end data.insert(ptr, value); }

Setting Behavior Using Template Arguments

  • Collection is defined by the type it stores, and by the type of its comparator void f(OrderedCollection<Employee, CompareByName>& c) {    ... } OrderedCollection<Employee, CompareBySalary> a; a.add(...); OrderedCollection<Employee, CompareByName> b; b.add(...); f(a); // Error - types don't match f(b); // OK - type is OK

Setting Behavior Using Template Arguments

  • Identify a default policy
  • Associated with a template parameter
  • STL algorithms use the less<T>, in the <functional> header file template<typename T> class less { bool operator()(const T& x, const T& y) const; } template<typename T> bool less<T>::operator()(const T& x, const T& y) const { return x < y; }

Setting Behavior Using Template Arguments

  • The less defines a function object
  • Invokes operator<
  • Specify trait as the default template argument template<typename T, typename CMP = less<T> > class OrderedCollection {    ... }
  • To use the default: OrderedCollection<int> data; // Will use the less than operator
