1. In specification inheritance:

    1. The interface is inherited
      The derived classes inherit the method signatures.
    2. The derived class inherits the base class implementation of the member functions.
      That is specialization inheritance.
    3. All derived concrete classes must override all pure-virtual base-class functions.
      Although you cannot create instances of abstract classes, you may create pointers to them.
    4. Allows a concrete class to decide whether to override a method.
      Specification requires concrete classes override all abstract members.
    5. May be combined with specialization inheritance.
      An ABC may have both virtual and pure-virtual member functions.
    6. A derived class which does not override all pure-virtual functions is also abstract.
      A derived class may override some abstract methods and leave it to its derived classes to override the others.
