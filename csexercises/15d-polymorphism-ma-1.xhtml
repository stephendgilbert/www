1. Given these classes:

    class Question { };
class ChoiceQuestion : public Question
{
public:
   void addChoice(const string& ans, bool correct);
};

    Which of the following statements are legal?

    1. ChoiceQuestion q = . . .;
Question a = q;
      This is legal, but a does not contain the choices. They have been sliced away.
    2. Question a = . . .;
ChoiceQuestion q = a;
      You cannot assign a base class object to a derived class object.
    3. ChoiceQuestion q = . . .;
Question a = q;
a.addChoice("Yes", true);
      a is a Question and does not have an addChoice member function.
    4. ChoiceQuestion q = . . .;
q.addChoice("No", false);
      q is a ChoiceQuestion and has an addChoice member function.
