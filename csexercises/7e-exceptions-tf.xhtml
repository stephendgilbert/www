1. Which of these statements are true?

    1. An uncaught exception in C++ is ignored.
      If an exception is thrown in C++ but not handled, the unexpected() handler is called, which results in program termination.
    2. If an exception is thrown in a function f(), but not handled there, the exception is sent (or propogated) to the function which called f().
    3. A circumstance in C++ where an exception is thrown must be an error.
      The exceptional circumstance is often an error, but this is not required.
    4. When an event occurs that cannot be managed locally, an exception may be thrown.
    5. Throwing the exception object transfers control and information generated locally to some calling function which manages the event, or, the program terminates.
    6. In C++ an exception object must be derived from the exception class defined in the standard library.
      An exception object in C++ can have any type in C++ including built-in types.
    7. A program can continue to run after an exception has been thrown and caught.
