1. What is true about an implementation (.cpp) file?

    1. It consists of function prototypes.
      The prototypes go in the interface file.
    2. It should always be surrounded by #ifndef...#endif header guards.
      You should never have header guards in an implementation file.
    3. You may define your functions here.
      Functions are defined in an implementation file.
    4. It may include a using namespace directive.
      This is fine in an implementation file.
    5. You may include headers from the standard library.
      This is required if you use a library type in your implementation.
    6. You may define global variables.
      Variables may be defined in an implementation.
    7. You must preface each type name used from the standard library with std::
      You may do that if you like. If you have a using namespace directive in your file, then you do not need to.
    8. string f() { return "hello"; } could appear.
      This is a function definition which is legal. You do not need to qualify the string type.
