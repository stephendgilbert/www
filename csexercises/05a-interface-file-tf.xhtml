1. Which of these statements are true about an interface (header) file?

    1. It may include function prototypes.
      A prototype is a declaration which is allowed.
    2. It should always be surrounded by #ifndef...#endif header guards.
      This ensures that a header file is only included once when you compile.
    3. You may define functions in a header file.
      Functions may be declared but not defined.
    4. It may include a using namespace directive
      You may not use namespace directives in header files.
    5. You may include headers from the standard library.
      This is required if you use a library type in a prototype.
    6. You may define global variables.
      Variables may not be defined in a header file.
    7. You may define global constants.
      Although variable definitions are prohibited, constant definitions are permitted.
    8. You must preface each type name used in your prototypes with std::
      Only library types are in the standard namespace. Primitive types are not.
    9. string f(); could appear in a header file.
      Library types must be fully qualified You would need to write std::string f(); in a header file.
