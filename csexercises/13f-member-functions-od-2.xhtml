1. It is a common beginner’s error to add types to assignment statements, thereby changing them into local variable declarations. Consider this example from a CashRegister user-defined type which has the data members totalPrice and payment:

    double CashRegister::giveChange()
{
   double change = payment - totalPrice; ➊ ➋
   double totalPrice = 0; ➌ ➍ // ERROR! This declares a local variable.
   double payment = 0; ➎ ➏ // ERROR! The data member is not updated.
   return change;
} ➐

    Trace through the function with the object shown and observe that the local variables are set to zero. Then they are removed when the function exits. The state of the CashRegister object is not changed.
