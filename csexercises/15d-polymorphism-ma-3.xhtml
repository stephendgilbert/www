1. Consider these classes:

    class Question
{
public:
   Question();
   void setText(const string& question_text);
   void setAnswer(const string& correct_response);
   virtual bool checkAnswer(const string& response) const;
   virtual void display() const;
private:
   string m_text;
   string m_answer;
};

class ChoiceQuestion : public Question
{
public:
   ChoiceQuestion();
   void addChoice(const string& choice, bool correct);
   virtual void display() const;
private:
   vector<string> m_choices;
};

    Which of the following statements is true?

    1. The display function in Question should not have been declared as virtual.
      This function is overridden in ChoiceQuestion.
    2. The checkAnswer function in Question should not have been declared as virtual.
      This function is not overridden in ChoiceQuestion, but another derived class may want to override it (for example, to allow for roundoff or spelling errors).
    3. The addChoice function in ChoiceQuestion should have been declared as virtual.
      No class derives from ChoiceQuestion, and if one did, it probably would not want to override this function.
    4. The display function in ChoiceQuestion should not have been declared as virtual.
      It is not required to declare the overriding method as virtual, but many programmers do so for clarity.
