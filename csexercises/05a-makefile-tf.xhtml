1. What is true about a makefile?

    1. It contains instructions for building your program.
    2. It can use variables from your .cpp file to control the building process.
      You may have makefile variables (macros) to control the build, but you cannot use C++ variables.
    3. It may build several different versions of a program.
      These are called targets. You may build a debug version and a release version.
    4. It may contain instructions that don't build a program.
      These are called pseudo targets. An example is make test.
    5. Targets are files required to build a dependency.
      The opposite: dependencies are files required to build a target.
    6. Targets and dependencies are separated by a colon (:).
      Example: $(EXE): $(OBJS).
    7. Each action just be indented by 4 spaces.
      You must use a tab character, NOT spaces in a makefile.
