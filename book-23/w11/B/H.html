The array & span Library Types

C++11 added the std::array library type in an attempt to create a safer array, without adding any of the overhead of the vector class. The new array type allows range-checked access to it's elements, knows the number of elements it contains, allows assignment, and comparison, just like vector.

Here are some examples that you can compare with the previous page, which shows the difference between the built-in arrays, and the array type 

array<int, 3> a1{1, 2, 3}, a2;
a2 = a1;                    // 1. Fine
a2[0] = a1[0];              // 2. Fine
a2 = {3, 4, 5};             // 3. Fine
if (a1 == a2) . . .         // 4. Fine
cout << a1.at(3);     // 5. throws an exception

There is one downside of using std::array instead of the built-in C-style array. When you pass the new std::array to a function, your function must specify the size of the array it is going to operate on, like this: 

void print(const string& desc, const array<int, 3>& a);

The print() function show here will only work on int arrays with three elements. You can fix this shortcoming by writing function templates, or, if you can use C++20, using the new span type.
